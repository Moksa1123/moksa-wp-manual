# 05.05 (實務) 提升轉換率的結帳頁面優化 (Checkout Field Editor)

嗨，歡迎來到 MoksaWP 網站設計與進階客製化課程。

在電商的世界裡，結帳頁面是顧客旅程的最後一哩路，也是最容易發生「棄單」的地方。一個複雜、多餘、不直覺的結帳流程，會讓顧客失去耐心，最終放棄購買。這堂課，我們將學習如何使用 **Checkout Field Editor (結帳欄位編輯器)** 這項強大的工具，來優化 WooCommerce 的結帳頁面，移除不必要的摩擦力，從而有效提升網站的轉換率。

我們的目標很明確：**讓顧客用最少的時間、填寫最少的資料，完成結帳。**

---

### 為什麼需要優化結帳頁面？

在開始實作之前，讓我們先建立一個重要的觀念。預設的 WooCommerce 結帳欄位是針對全球市場設計的，包含了許多在台灣的電商環境中並非必要的欄位，例如：

*   **公司名稱 (Company name):** 對 B2C 零售網站來說，絕大多數顧客都用不到。
*   **地址 2 (Apartment, suite, unit, etc.):** 台灣的地址通常一行就能寫完。
*   **郵遞區號 (Postcode / ZIP):** 雖然有用，但我們的物流外掛 **RY Tools** 可以根據縣市和鄉鎮市區自動帶入，不一定需要顧客手動填寫。

每多一個欄位，就等於增加一個讓顧客分心或放棄的機會。透過優化，我們可以打造一個專為台灣使用者設計，乾淨、快速、且高轉換率的結帳體驗。

### 認識工具：Checkout Field Editor for WooCommerce

市面上有許多結帳欄位編輯外掛，Moksa 團隊推薦由 **ThemeHigh** 開發的 **Checkout Field Editor for WooCommerce**。它功能完整，介面直觀，並且有免費版本可以使用。

**步驟 1：安裝與啟用外掛**

1.  前往 WordPress 後台 > **外掛** > **安裝外掛**。
2.  在搜尋框中輸入「**Checkout Field Editor for WooCommerce**」。
3.  找到由 **ThemeHigh** 開發的外掛，點擊 **立即安裝** 並 **啟用**。

### (實作一) 移除不必要的欄位

這是優化的第一步，也是最關鍵的一步。我們要大刀闊斧地移除那些對我們的業務模式沒有幫助的欄位。

**步驟 1：進入設定頁面**

前往 **WooCommerce** > **Checkout Form**。你會看到三個頁籤：**Billing Fields (帳單欄位)**, **Shipping Fields (運送欄位)**, 和 **Additional Fields (附加欄位)**。

` `
*(Checkout Field Editor 的主設定介面)*

**步驟 2：編輯帳單欄位 (Billing Fields)**

帳單欄位是結帳的核心。讓我們來一一檢視並決定去留。

1.  **Company name (公司名稱):** 如果你的主要客群是個人消費者，這個欄位就可以移除。找到 `billing_company` 這一行，取消勾選 **Enabled** 欄位下的核取方塊。
2.  **Country / Region (國家/地區):** 如果你只運送商品到台灣，這個欄位可以預設為「台灣」並隱藏起來。點擊 `billing_country` 旁的 **Edit** 按鈕，在 **Display Options** 區塊，將 **Default Value** 設為 `Taiwan`。然後回到主畫面，取消勾選 `billing_country` 的 **Enabled**。
3.  **Street address (地址):** `billing_address_1` 是必要的，但 `billing_address_2` (地址 2) 幾乎用不到。找到 `billing_address_2` 並取消勾選 **Enabled**。
4.  **Postcode / ZIP (郵遞區號):** **Moksa 技巧：** 我們的物流外掛 **RY Tools** 可以自動處理郵遞區號，為了簡化流程，我們可以考慮移除這個手動填寫的欄位。找到 `billing_postcode`，取消勾選 **Enabled**。

**步驟 3：編輯運送欄位 (Shipping Fields)**

如果你啟用了「運送到不同的地址」，這裡的欄位也需要一併調整。通常，我們會讓運送欄位的設定與帳單欄位保持一致。重複上述步驟，將 `shipping_company`, `shipping_country`, `shipping_address_2`, `shipping_postcode` 等欄位取消啟用。

**步驟 4：儲存變更**

完成以上設定後，務必點擊頁面底部的 **Save changes** 按鈕。

` `
*(完成欄位簡化後的設定畫面)*

> **⚠️ 重要提醒：哪些欄位絕對不能亂動？**
> *   **核心顧客資訊:** `billing_first_name` (名), `billing_last_name` (姓), `billing_phone` (電話), `billing_email` (電子郵件)。這些是訂單成立與後續聯絡的基礎。
> *   **物流必要資訊:** `billing_address_1`, `billing_city`, `billing_state`。這些是 **RY Tools** 計算運費和產生託運單的關鍵，特別是超商取貨的運作也依賴這些基礎欄位。
> *   **金流/發票欄位:** 由**綠界金流**或**綠界發票**外掛自動產生的欄位（例如：發票類型、載具、統一編號）。這些欄位通常有特定的 `name`，請不要輕易移除或修改，否則會導致金流或發票功能異常。

### (實作二) 調整欄位順序與文字

移除欄位後，我們可以進一步調整現有欄位的顯示方式，讓它更符合台灣用戶的習慣。

**步驟 1：調整欄位順序**

直接用滑鼠拖放欄位即可調整它們的上下順序。例如，你可以把「電話」和「電子郵件」向上移動，放在「姓名」的下方，讓聯絡資訊更集中。

**步驟 2：修改欄位標籤 (Label) 與預留位置 (Placeholder)**

預設的欄位文字是英文翻譯過來的，我們可以把它修改得更口語化、更清晰。

1.  找到 `billing_first_name`，點擊 **Edit**。
2.  將 **Label** 修改為「**收件人姓名**」。(Moksa 技巧：你也可以將姓、名合併成一個欄位，但考量到金流與物流的資料格式，我們建議維持分開)
3.  找到 `billing_address_1`，點擊 **Edit**。
4.  將 **Label** 修改為「**詳細地址**」。
5.  將 **Placeholder** 修改為「**請填寫完整路名、巷弄、門牌號碼**」。
6.  找到 `billing_state` (省/州)，這在台灣對應的是「縣市」。將 **Label** 修改為「**縣市**」。
7.  找到 `billing_city` (城鎮/鄉鎮市區)，將 **Label** 修改為「**鄉鎮市區**」。

*(編輯欄位 Label 和 Placeholder 的彈出視窗)*

完成後儲存，去前台結帳頁面看看，是不是變得清爽又好懂了呢？

### (實作三) 新增實用的自訂欄位

除了刪減，我們還可以增加一些有商業價值的自訂欄位，來蒐集更多顧客資訊或提供貼心服務。

**情境一：新增「希望到貨時段」**

如果你的宅配服務可以指定到貨時段，提供這個選項會非常加分。

**步驟 1：新增欄位**

在 **Billing Fields** 或 **Additional Fields** 頁籤下方，點擊 **Add new field** 按鈕。

**步驟 2：設定欄位屬性**

*   **Type:** 選擇 **Select** (下拉選單)。
*   **Name:** `preferred_delivery_time` (**Moksa 技巧：** 欄位名稱 **Name** 必須是英文、小寫，且不能重複，這是資料庫儲存的 key)。
*   **Label:** **希望到貨時段**。
*   **Enabled:** 勾選。
*   **Required:** 根據你的業務需求決定是否必填。如果只是附加服務，可以不勾選。

**步驟 3：設定選項 (Options)**

在 **Options** 區塊，點擊 **Add option** 來新增選項。

*   **Option key:** `morning` | **Option text:** `上午 (09:00-13:00)`
*   **Option key:** `afternoon` | **Option text:** `下午 (14:00-18:00)`
*   **Option key:** `anytime` | **Option text:** `不指定`

` `
*(設定「希望到貨時段」的下拉選單欄位)*

**步驟 4：儲存並查看**

儲存後，這個欄位就會出現在結帳頁面了！顧客填寫的資訊，你可以在後台的訂單詳情頁面看到。

**情境二：新增「從何處得知我們？」(行銷追蹤)**

這個欄位對於分析行銷管道成效非常有幫助。

*   **Type:** **Select** (下拉選單) 或 **Radio** (單選按鈕)。
*   **Name:** `marketing_source`。
*   **Label:** **請問您是從何處得知我們的呢？**
*   **Options:** Facebook, Google 搜尋, Instagram, 朋友推薦, 其他。

> **🚀 Moksa 進階技巧：整合 FluentCRM**
> 你在結帳頁面新增的任何自訂欄位，都可以與 **FluentCRM** 進行深度整合！
>
> 1.  在 FluentCRM 中，前往 **Contacts** > **Custom Fields**，建立一個對應的自訂欄位 (例如：Marketing Source)。
> 2.  接著，在 FluentCRM 的自動化設定中，觸發器選擇「WooCommerce Order Created」。
> 3.  在後續的動作中，選擇「Update Contact Property」，你就可以將結帳時 `marketing_source` 這個欄位的值，同步更新到顧客在 FluentCRM 中的「Marketing Source」欄位。
>
> 這樣一來，你就可以根據顧客來源，將他們加入不同的標籤 (Tag) 或名單 (List)，並發送個人化的行銷 Email。這就是數據驅動行銷的威力！

### 總結與課後作業

今天我們學會了如何透過 **Checkout Field Editor** 這個強大的外掛，對 WooCommerce 的結帳頁面進行徹底的優化。

**重點回顧：**

1.  **移除原則：** 大膽移除不必要的欄位，特別是針對台灣市場的「公司名稱」、「地址二」等。
2.  **保留原則：** 絕對不要移除顧客核心資訊、物流必要欄位，以及金流/發票外掛的專用欄位。
3.  **優化文字：** 將欄位標籤和提示文字修改得更符合在地用語，降低顧客的理解成本。
4.  **增加價值：** 策略性地新增自訂欄位，如「希望到貨時段」或「行銷來源」，不僅能提升服務品質，更能蒐集寶貴的商業數據。
5.  **測試、測試、再測試：** 每次修改完結帳頁面後，**務必**自己親自下單測試整個流程，從填寫資料、選擇金物流、付款到最後收到訂單通知，確保一切運作順暢。

**課後作業：**

請根據你自己的網站業務模式，動手優化你的結帳頁面。
1.  至少移除 2 個預設欄位。
2.  修改至少 3 個欄位的標籤 (Label) 文字。
3.  新增 1 個對你的生意有幫助的自訂欄位。

完成後，試著用訪客身分下單看看，感受一下流程是否真的變得更順暢了。我們下一堂課見！
