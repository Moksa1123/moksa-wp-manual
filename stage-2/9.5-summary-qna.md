## 課程重點回顧

讓我們快速回顧在這 40 小時中，我們共同掌握的核心知識與技能。

### 1. 現代化的設計工具與思維

我們拋棄了傳統、笨重的頁面編輯器，全面擁抱以 Gutenberg 核心為基礎的現代化工具。

*   **Kadence Blocks:** 你學會了如何利用 Kadence Blocks 打造出輕量、高效能且極具彈性的頁面。我們深入探討了它的 **Row Layout**、**Advanced Heading** 以及強大的**全域色彩 (Global Colors)** 與字型系統，讓你能夠建立一致且易於維護的設計系統。
*   **Elementor:** 我們也探索了 Elementor 的強大生態系與視覺化工作流程，特別是它的 **Theme Builder** 與 **Loop Grid** 功能，讓你能夠完全客製化網站的每一個角落，從 Header、Footer 到文章封存頁面。
*   **設計思維：** 更重要的是，你學會了**如何選擇工具**。**Moksa 技巧：** 新專案優先考慮以 Kadence 為主的區塊編輯器，以獲得最佳效能與未來相容性；而 Elementor 則是你應對既有專案或需要龐大生態系支援時的強力武器。

### 2. 進階版面與動態內容

靜態網站已無法滿足多樣化的商業需求。你已經掌握了讓網站「活」起來的關鍵技術。

*   **CPT (Custom Post Types):** 你不再受限於 WordPress 內建的「文章」與「頁面」。你學會了如何建立「作品集」、「房地產物件」、「課程列表」等自訂內容類型，讓網站結構更清晰、管理更方便。
*   **ACF (Advanced Custom Fields):** 這是你最強大的武器之一。透過 ACF，你可以為任何內容類型添加結構化的自訂欄位，如：價格、規格、日期、地點等。
*   **動態內容整合：** 你學會了如何將 CPT 與 ACF 的資料，透過 Kadence Blocks Pro 或 Elementor Pro 的 **Dynamic Content** 功能，優雅地呈現在前端頁面，打造出複雜的目錄網站、作品集網站，而無需撰寫任何一行程式碼。

### 3. 網站效能與使用者體驗 (UX)

一個再漂亮的網站，如果載入緩慢，都將失去價值。我們從一開始就將效能優化融入在每一個開發環節。

*   **圖片優化：** 使用 **ShortPixel** 自動壓縮與轉換 WebP 格式，從源頭解決圖片過大的問題。
*   **快取機制：** 設定 **WP Rocket**，學會了頁面快取、延遲載入 JavaScript、CSS 優化等關鍵技術，大幅提升網站載入速度與 PageSpeed Insights 分數。
*   **主機選擇：** 我們也探討了優質主機對於網站效能的根本性影響，讓你理解為何投資一個好的主機是絕對必要的。

### 4. 開發與維護的最佳實踐

專業，體現在你的工作流程中。你學會了業界標準的開發與維護流程。

*   **本地開發：** 使用 **LocalWP** 在你的電腦上建立安全、快速的開發環境，告別直接在線上主機修改網站的風險。
*   **版本控制：** 初步接觸了 **Git** 的概念，理解版本控制對於專案管理與團隊協作的重要性。
*   **網站維護：**
    *   **備份：** 設定 **UpdraftPlus** 定期自動備份，確保你的心血結晶永遠安全。
    *   **安全：** 安裝 **Wordfence**，了解基本的防火牆與惡意軟體掃描設定，保護網站免於常見威脅。
*   **偵錯除錯：** 你學會了使用 **Query Monitor** 和 `WP_DEBUG` 模式來分析網站的效能瓶頸與錯誤來源，讓你面對問題時不再束手無策。

## 常見問題 Q&A

**Q1: 學完這堂課，我真的可以開始接案了嗎？**

**A:** **絕對可以。** 你已經具備了建立一個專業、高效能且易於維護的客製化網站所需的核心技能。

**Moksa 技巧：** 建議從規模較小的專案開始，例如：形象網站、部落格。這能幫助你建立信心、累積作品，並熟悉與客戶溝通的流程。記住，**溝通能力、專案管理能力與你的技術能力同等重要**。誠實地告知客戶你的經驗，並提供合理的報價，是建立長期信譽的關鍵。

**Q2: 面對專案時，我應該選擇 Kadence 還是 Elementor？**

**A:** 這個問題沒有標準答案，取決於專案需求。以下是我們的決策框架：

*   **優先選擇 Kadence Blocks 的情境：**
    *   全新的專案。
    *   對網站載入速度有極致要求的客戶。
    *   網站內容結構清晰，主要依賴文章、頁面與 CPT。
    *   你希望更貼近 WordPress 原生體驗，未來維護成本較低。
*   **優先選擇 Elementor 的情境：**
    *   客戶的既有網站就是用 Elementor 建立的。
    *   專案需要大量使用 Elementor 生態系獨有的第三方附加元件 (Add-ons)。
    *   設計稿極度複雜，需要大量的絕對定位、動態特效，Elementor 的視覺化操作可能更直覺。

**Q3: 我遇到了外掛衝突或網站出現「白畫面」 (WSOD)，該怎麼辦？**

**A:** 不要慌張，這是一個必經的過程。請遵循標準的偵錯流程 (SOP)：

*   **步驟 1：啟用偵錯模式**
    *   透過 FTP 或主機的檔案管理員，編輯網站根目錄的 `wp-config.php` 檔案，將 `define( 'WP_DEBUG', false );` 改為 `true`。
    *   `define( 'WP_DEBUG', true );`
    *   `define( 'WP_DEBUG_LOG', true );`
    *   `define( 'WP_DEBUG_DISPLAY', false );`
    *   這樣會將錯誤訊息記錄在 `wp-content/debug.log` 檔案中，而不會直接顯示在網站前台。

*   **步驟 2：檢查錯誤日誌**
    *   查看 `debug.log` 檔案，錯誤訊息通常會明確指出是哪個外掛或主題的哪一行程式碼出了問題。

*   **步驟 3：停用外掛**
    *   如果無法登入後台，透過 FTP 或檔案管理員，將 `wp-content/plugins` 資料夾的名稱暫時改為 `plugins_old`。這會強制停用所有外掛。
    *   如果網站恢復正常，就表示是外掛衝突。將資料夾名稱改回來，然後到後台逐一啟用外掛，直到找到引發問題的元兇。

*   **步驟 4：切換預設主題**
    *   如果停用外掛後問題仍在，可以嘗試用類似的方法，將你正在使用的主題資料夾改名，WordPress 會自動切換回預設主題（例如 Twenty Twenty-Four）。

**Q4: 我對程式碼更有興趣，下一步該學習什麼？**

**A:** 非常棒！這將為你開啟一扇新的大門。建議的學習路徑如下：

1.  **PHP 基礎：** 深入學習 PHP，特別是與 WordPress 相關的 Hooks (Actions & Filters)，這是客製化 WordPress 功能的核心。
2.  **JavaScript (ES6+)：** 精通現代 JavaScript 是未來 WordPress 開發的基礎。
3.  **React.js：** Gutenberg 編輯器是基於 React.js 開發的。學習 React 是開發自訂區塊的必備技能。
4.  **WP-CLI：** 學習使用命令列工具管理 WordPress，能大幅提升你的工作效率。
5.  **REST API：** 了解如何透過 WordPress REST API 串接資料，這是實現無頭 (Headless) WordPress 應用的基礎。
