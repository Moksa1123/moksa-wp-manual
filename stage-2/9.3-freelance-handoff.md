# 09.03 (實務) 如何「交站」給客戶 (製作手冊、教育訓練)

大家好，我是 Moksa 的首席課程導師。

網站開發完成並不是專案的終點，而是一個新階段的開始。一個**專業、順暢的「交站」流程**，是確保客戶能夠順利接管網站、發揮其最大價值的關鍵，更是我們專業服務的最終體現。一個好的交站流程，不僅能提升客戶滿意度，更能大幅減少日後不必要的售後支援成本。

這堂課，我們將完整拆解 Moksa 團隊標準化的交站 SOP，包含交站前的準備、如何撰寫一份客戶看得懂的操作手冊，以及如何執行一場有效的教育訓練。

---

## 1. 交站前的準備清單 (Checklist)

在與客戶約定教育訓練時間之前，務必完成以下準備工作，確保網站處於最佳狀態。這不僅是專業的表現，也能避免在訓練過程中遇到突發狀況。

*   **[ ] 最終功能測試：**
    *   **前台功能：** 重新走一次完整的購物流程（從商品頁 -> 加入購物車 -> 結帳 -> 付款）。使用**綠界測試信用卡**與**虛擬 ATM** 都測試一遍。
    *   **後台功能：** 確認訂單狀態是否會隨付款結果自動更新、確認 **RY Tools** 物流拋單功能正常、確認**綠界電子發票**是否能正常開立。
    *   **RWD 響應式設計：** 使用瀏覽器的開發者工具，模擬手機、平板等不同裝置，確保版面沒有跑版。
    *   **表單測試：** 再次檢查所有聯絡表單、訂閱表單是否能正常收到信件。

*   **[ ] 效能與 SEO 檢查：**
    *   執行 **WP Rocket** 的快取清除與預載入 (Preload)，確保快取已是最新版本。
    *   使用 Google PageSpeed Insights 或 GTmetrix 進行最後的速度檢測。
    *   登入 Google Search Console，確認 Sitemap 已提交且沒有重大索引錯誤。
    *   使用 **Rank Math SEO** 檢查重點頁面 (首頁、關於我們、核心商品分類) 的 SEO Title 和 Meta Description 是否都已設定。

*   **[ ] 網站清理與整理：**
    *   刪除所有測試用的商品、訂單、文章、頁面與使用者。
    *   停用並刪除開發過程中使用的偵錯外掛，例如 Query Monitor。
    *   刪除所有用不到的佈景主題與外掛，保持後台乾淨。

*   **[ ] 建立客戶專用帳號：**
    *   **Moksa 技巧：** **絕對不要**直接交付「管理員 (Administrator)」權限的帳號給客戶的日常操作人員。
    *   **步驟 1：** 前往 WordPress 後台 >「使用者」>「新增使用者」。
    *   **步驟 2：** 建立一個**角色**為「**商店管理員 (Shop Manager)**」的帳號給客戶。這個角色擁有管理 WooCommerce 所有功能的權限，但無法修改佈景主題、安裝/刪除外掛，相對安全。
    *   **步驟 3：** 如果客戶需要管理文章或頁面，可以考慮建立「**編輯 (Editor)**」權限的帳號。

*   **[ ] 完整網站備份：**
    *   使用 **UpdraftPlus** 進行一次完整的手動備份（包含資料庫與所有檔案）。
    *   將備份檔案下載到你的電腦或雲端硬碟中，這份備份是「交站初始版本」，非常重要。

*   **[ ] 彙整帳號密碼清單：**
    *   將以下所有相關服務的帳號、密碼、重要金鑰 (API Key) 整理在一份文件中（建議使用密碼管理工具如 1Password 或 Bitwarden 分享）：
        *   WordPress 管理員帳號、商店管理員帳號
        *   主機商 (Hosting) 登入資訊
        *   網域 (Domain) 管理後台資訊
        *   綠界科技 (ECPay) 廠商後台帳密、API Key
        *   LINE Pay 合作商店後台帳密
        *   Google Analytics (GA), Google Search Console 權限 (需請客戶將你加入)
        *   其他第三方服務 (例如：FluentCRM, 電子報服務等)

## 2. 撰寫客製化網站操作手冊

一份好的操作手冊應該是圖文並茂、淺顯易懂的。不要假設客戶懂任何專有名詞。我們的目標是讓客戶的員工，即使沒有任何 WordPress 經驗，也能按圖索驥完成日常工作。

**Moksa 技巧：** 我們推薦使用 Google Docs 或 Notion 來製作手冊，方便線上共享、評論與更新。完成後可以匯出成 PDF 檔交付給客戶。

### 手冊架構建議

`[截圖：展示一份專業手冊的封面，包含客戶 Logo 與網站名稱]`

#### **第一部分：網站基礎資訊**

這部分是緊急情況下最重要的資訊。

*   **1.1 網站後台登入資訊**
    *   後台網址：`https://你的網域.com/wp-admin`
    *   商店管理員帳號/密碼
    *   管理員帳號/密碼 (可選擇性提供，或告知客戶有需要時再提供)
*   **1.2 主機與網域資訊**
    *   存放「帳號密碼清單」的安全連結。
*   **1.3 重要外掛授權碼 (License Keys)**
    *   列出需要付費續約以獲得更新的重要外掛，例如：WP Rocket, Elementor Pro, Advanced Coupons 等，並標示其到期日。

#### **第二部分：日常營運 SOP**

這部分是客戶最常使用的功能，需要寫得最詳細。

*   **2.1 如何上架一個「簡單商品」**
    *   **步驟 1：** 進入「商品」>「新增商品」。
    *   **步驟 2：** 填寫「**商品名稱**」與「**詳細描述**」。
    *   **步驟 3：** 在右側欄位設定「**商品分類**」與「**商品標籤**」。
    *   **步驟 4：** 上傳「**商品圖片**」(主圖) 與「**商品相簿**」(附加圖片)。**Moksa 技巧：** 強調圖片上傳前應使用 ShortPixel 進行壓縮，或在上傳後檢查壓縮狀態。
    *   **步驟 5：** 在下方的「商品資料」區塊，設定「**一般**」分頁的「**價格**」與「**特價**」。
    *   **步驟 6：** 設定「**庫存**」分頁的 SKU 與庫存管理。
    *   **步驟 7：** 填寫「**商品簡短描述**」。
    *   **步驟 8：** 點擊「**發佈**」。
*   **2.2 如何管理訂單 (標準出貨流程)**
    *   **步驟 1：** 進入「WooCommerce」>「訂單」，查看狀態為「**處理中**」的訂單，這代表客戶已付款成功。
    *   **步驟 2：** 勾選要出貨的訂單，使用 **RY Tools** 的「**下載標籤 (中華郵政/超商)**」功能，批次拋轉出貨資料給物流系統。
    *   `[截圖：展示 RY Tools 批次拋單的操作介面]`
    *   **步驟 3：** 從物流系統後台 (例如：綠界 C2C 平台) 列印託運單，並貼在包裹上。
    *   **步驟 4：** 將包裹寄出後，回到訂單列表，勾選已寄出的訂單，並將訂單狀態手動更新為「**已完成**」。此時，系統會自動寄發「已完成通知信」給客戶。
    *   **Moksa 技巧：** 提醒客戶，**綠界電子發票**外掛通常設定在訂單狀態轉為「已完成」時，自動開立並寄送發票給顧客。
*   **2.3 如何新增/編輯頁面與文章**
    *   簡單說明如何使用 **Kadence Blocks** 或 **Elementor** 編輯器來修改「關於我們」、「聯絡我們」等靜態頁面。
    *   教學如何新增一篇部落格文章，並設定其 **Rank Math SEO** 標題與描述。

#### **第三部分：行銷活動操作**

*   **3.1 如何建立優惠券**
    *   教學 WooCommerce 內建的「折扣 %」、「固定金額折扣」優惠券。
    *   **Moksa 技巧：** 特別教學 **Advanced Coupons** 的獨特功能，例如如何建立「**BOGO (買一送一)**」或「**網址優惠券 (URL Coupons)**」。
*   **3.2 如何發送行銷 Email**
    *   教學如何使用 **FluentCRM** 建立 Email 範本，並篩選特定標籤 (Tag) 的客戶來發送電子報。
*   **3.3 如何與客戶 LINE 互動**
    *   教學如何登入 **OrderChatz** 後台，查看透過 LINE 傳來的客服訊息，並設定 AI 自動回覆。

## 3. 執行線上/實體教育訓練

有了詳盡的手冊當作教材，教育訓練就會事半功倍。

*   **形式：** 建議使用 Google Meet 或 Zoom 進行線上會議，方便螢幕分享與**全程錄影**。
*   **時間：** 建議 1.5 ~ 2 小時，避免資訊量過載。
*   **議程：**
    1.  **開場 (10分鐘):** 再次介紹網站的整體架構與前台功能亮點。
    2.  **核心功能演練 (60分鐘):**
        *   **親自操作一次**手冊第二部分的「日常營運 SOP」。從上架商品開始，然後模擬客戶下單，再到後台處理這筆訂單（拋單、更新狀態）。
        *   **Moksa 技巧：** **請客戶跟著你的步驟，實際操作一遍**。讓他們親手新增一個商品、修改一筆訂單，肌肉記憶的效果遠勝於單純觀看。
        *   演練一個行銷活動，例如建立一張 BOGO 優惠券並測試。
    3.  **Q&A 與補充 (20分鐘):** 解答客戶提出的問題。
    4.  **總結與後續步驟 (5分鐘):** 告知客戶你會將**今天的訓練錄影**、**網站操作手冊**、**帳號密碼清單**一併寄送給他們，並說明交站後的支援期限（例如：14 天的 Email 諮詢期）。

## 4. 交站後的支援與維護

交站不代表關係的結束，而是長期合作的開始。

*   **定義支援範圍：** 在合約或交站信件中，明確定義交站後提供的「免費支援」範圍與期限。通常包含：
    *   **期限：** 14 天或 30 天。
    *   **範圍：** 針對「現有功能」的操作疑問提供解答、修正因我方疏失造成的 Bug。
    *   **不包含：** 新功能開發、版面大幅修改、因客戶自行安裝外掛或操作不當造成的問題。
*   **提供維護方案：**
    *   **Moksa 技巧：** 在交站時，主動提供「**網站代管維護方案**」。這是一個創造持續性收入的好機會。
    *   說明維護方案的價值：定期**外掛/核心更新**、**安全性監控 (Wordfence)**、**效能監控**、**定期備份 (UpdraftPlus)**，讓客戶可以專注於他們的本業，由我們來處理技術問題。
*   **交付文件：**
    *   將以下檔案打包，透過 Email 或客戶習慣的通訊軟體正式交付：
        1.  網站操作手冊 (PDF)
        2.  教育訓練錄影檔 (上傳到 YouTube 私人影片或雲端硬碟)
        3.  帳號密碼清單的安全分享連結

完成以上所有步驟，你就完成了一次專業、完整、讓客戶安心的交站流程。這不僅為專案畫下完美句點，也為未來的合作奠定了信任的基礎。
