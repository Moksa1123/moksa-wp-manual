# 08.04 (實作) 建立「社團 (Groups)」

嗨，歡迎來到這個單元！在前面的課程中，我們已經為網站建立了核心的社群功能。現在，我們要來探討 BuddyBoss 中一個非常強大且關鍵的組件 — **社團 (Social Groups)**。

社團功能允許你或你的會員建立基於共同興趣、課程、專案或任何主題的微型社群。這不僅能大幅提升會員的參與度和黏著度，更是將你的網站從一個單純的內容平台，轉變為一個活躍互動社群的關鍵。

在本章節中，我們將學習如何啟用並設定社團功能，並實際從後台建立一個與 LearnDash 課程綁定的專屬私密社團。

---

### 前置作業：啟用社團 (Groups) 組件

在我們開始建立社團之前，必須先確認 BuddyBoss 的「社團」組件已經是啟用狀態。

*   **步驟 1：** 前往 WordPress 後台，在左側選單找到 `BuddyBoss` -> `Components`。
*   **步驟 2：** 在元件列表中，找到 `Social Groups` 這個項目。
*   **步驟 3：** 如果它尚未啟用，請點擊右側的 `Activate` 按鈕。啟用後，它會顯示為 `Deactivate`。

``
*(截圖：BuddyBoss Components 頁面，並用紅框圈起 Social Groups 組件)*

**Moksa 技巧：** BuddyBoss 採用模組化設計，你可以只啟用你需要的功能。如果你的網站暫時不需要社團功能，可以將其停用，以節省系統資源。

### 步驟一：全域社團設定 (Global Group Settings)

啟用組件後，我們需要對社團功能進行一些全域性的設定，這會影響到網站上所有社團的運作方式。

*   **步驟 1：** 前往 `BuddyBoss` -> `Settings` -> `Groups` 頁籤。
*   **步驟 2：** 在這裡，你會看到幾個重要的設定選項，讓我們來逐一了解：
    *   **Group Creation：** 這個選項決定了「誰」可以建立社團。你可以設定為**所有會員 (All members)**、**網站管理員與社團管理員 (Site admins and moderators only)**。
    *   **Group Types：** 這是非常實用的分類功能。你可以預先建立好不同的「社團類型」，例如：「課程專屬」、「興趣同好」、「地區分會」等。這樣在建立社團時，就可以為其指定一個類型，方便會員篩選和尋找。我們先點擊 `Add New Type`，建立一個名為「**課程專屬社團**」的類型。
    *   **REST API：** 保持預設的 `Enable REST API` 即可，這對於未來與 App 或其他服務整合很重要。

``
*(截圖：BuddyBoss 的 Groups Settings 頁面，顯示 Group Creation 和 Group Types 的設定)*

**Moksa 技巧：** 對於剛起步的社群網站，我們**強烈建議**將 `Group Creation` 設定為「**網站管理員與社團管理員 (Site admins and moderators only)**」。這樣可以確保初期社團的品質，避免會員隨意建立大量低品質或無人管理的「殭屍社團」。等到社群成熟且活躍後，再考慮開放給所有會員。

### 步驟二：從後台建立第一個社團

現在，讓我們來實際操作，建立一個專為某個課程設計的私密討論社團。

*   **步驟 1：** 前往 WordPress 後台，在左側選單找到 `Groups` -> `Add New`。

``
*(截圖：WordPress 後台，點擊 Groups -> Add New 的畫面)*

*   **步驟 2：** 進入新增社團頁面後，你會看到多個欄位和頁籤，請依照以下說明填寫：

    **在 `Details` 頁籤中：**
    *   **社團名稱 (Group Name):** **必填**。輸入一個清晰的名稱，例如：「**MoksaWP 網站設計實戰班 - 專屬討論區**」。
    *   **社團描述 (Group Description):** **必填**。詳細說明這個社團的用途、規則和目標。這段描述會顯示在社團的主頁。
    *   **社團類型 (Group Type):** 選擇我們剛剛建立的「**課程專屬社團**」。

``
*(截圖：新增社團的 Details 頁籤，填寫名稱、描述和類型)*

*   **步驟 3：** 點擊上方的 `Settings` 頁籤，進行社團隱私與功能的設定。這是**最重要**的步驟。
    *   **隱私選項 (Privacy Options):**
        *   **This is a public group：** 公開社團。任何人都能看到社團內容和成員，並且可以自由加入。適合興趣導向的社團。
        *   **This is a private group：** 私密社團。任何人都能在社團目錄中看到這個社團，但必須申請並由管理員核准後才能加入及查看內容。**這是課程社團最常用的設定**。
        *   **This is a hidden group：** 隱藏社團。這個社團不會出現在公開的社團目錄中，只能透過邀請才能加入。適合內部員工、核心團隊或 VIP 會員。
        *   我們在此選擇「**This is a private group**」。
    *   **社團邀請 (Group Invitations):** 設定誰可以邀請新成員加入。對於私密社團，建議設定為「**All group members**」或「**Group admins and mods only**」。
    *   **活動摘要 (Activity Feeds):** 勾選 `Enable activity feed for this group`，讓成員可以在社團內發佈動態。

``
*(截圖：新增社團的 Settings 頁籤，特別圈起 Privacy Options)*

*   **步驟 4：** **Moksa 整合技巧：** 點擊上方的 `Courses` 頁籤。這個頁籤是 BuddyBoss 與 LearnDash 深度整合的體現。
    *   在 `Add Courses` 的搜尋框中，輸入你想綁定的課程名稱，例如「MoksaWP 網站設計實戰班」。
    *   選取該課程後，下方的 `Auto-join` 選項會出現。你可以設定當學員「**註冊課程 (Course is completed)**」或「**完成課程 (User is enrolled to the course)**」時，自動將他們加入這個社團。
    *   我們選擇「**User is enrolled to the course**」，這樣只要學員購買或註冊了這門課，就會自動成為社團成員，無需手動審核。

``
*(截圖：Courses 頁籤，顯示如何搜尋並綁定一個 LearnDash 課程，並設定自動加入)*

*   **步驟 5：** 其他頁籤如 `Forum` (串接 bbPress 論壇)、`Photos` (相簿功能)、`Albums` (影音相簿) 等，可以根據你的需求啟用。
*   **步驟 6：** 所有設定完成後，點擊右側的「**發佈 (Publish)**」按鈕。

恭喜你！你的第一個社團已經成功建立。

### 步驟三：從前台檢視與管理社團

社團的主要互動都是發生在網站前台。

*   **步驟 1：** 以前台登入的會員身份，點擊個人選單或網站導覽列中的「**社團 (Groups)**」連結。
*   **步驟 2：** 你會看到社團目錄頁面，你剛剛建立的「MoksaWP 網站設計實戰班 - 專屬討論區」應該會出現在列表中，並標示為「Private Group」。
*   **步驟 3：** 點擊進入社團頁面。你會看到一個類似 Facebook 社團的介面，包含了動態牆 (Activity)、成員 (Members)、邀請 (Invite) 等功能。身為管理員，你還會看到「**管理 (Manage)**」分頁，可以在此直接從前台修改社團設定、管理成員。

``
*(截圖：網站前台的單一社團頁面，展示動態牆、成員列表和管理分頁)*

### Moksa 總結與最佳實務

建立社團只是第一步，成功的社群需要持續的經營。

1.  **善用社團類型 (Group Types):** 隨著社團數量增加，良好的分類能幫助會員快速找到他們感興趣的內容。
2.  **明確的社團規則：** 在社團描述中清楚寫下社團規範，並指派「**社團管理員 (Moderators)**」協助維護討論品質。
3.  **主動發起討論：** 在社團初期，管理員應主動發起話題、舉辦活動 (例如：問答、投票)，引導成員參與互動。
4.  **整合行銷自動化：** 你可以結合 **FluentCRM**，設定自動化流程。例如，當有新成員透過 LearnDash 自動加入社團時，發送一封歡迎 Email，引導他們進行第一次發文或自我介紹。
5.  **搭配 LINE 推播：** 透過 **OrderChatz**，你可以針對購買特定課程 (並加入社團) 的學員，發送一則 LINE 訊息，提醒他們記得前往專屬社團參與討論，大幅提升觸及率與參與度。

社團功能是 BuddyBoss 的靈魂所在。靈活運用它，你就能為你的會員打造一個充滿歸屬感和價值的線上家園。在下一個單元，我們將探討如何進一步管理社團成員與權限。
