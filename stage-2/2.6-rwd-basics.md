# 02.06 (實務) RWD 響應式設計 (平板/手機)

嗨，歡迎來到 Moksa 網站設計的實務課程！在前幾堂課中，我們學會了如何使用頁面編輯器 (Kadence Blocks 或 Elementor) 來打造精美的電腦版 (Desktop) 網頁。然而，在現今這個時代，超過一半的網站流量都來自於手機與平板，因此，**確保網站在任何裝置上都能完美呈現**，不僅是專業的表現，更是影響使用者體驗與轉換率的關鍵。

這堂課，我們將深入探討 **RWD (Responsive Web Design) 響應式設計** 的核心觀念與實作技巧。你將學會如何使用編輯器內建的工具，針對平板和手機模式進行精準的版面、間距、文字大小調整，甚至隱藏或顯示特定元素，讓你的網站真正做到「一站多螢，完美適應」。

---

### RWD 的核心觀念

在我們動手實作之前，先來理解幾個 RWD 的核心觀念：

1.  **斷點 (Breakpoints):**
    *   這是 RWD 的基礎。頁面編輯器會預設幾個「螢幕寬度」的臨界點，例如 `1024px` 以下視為平板，`768px` 以下視為手機。當使用者的螢幕寬度跨越這些斷點時，我們設定的響應式樣式就會生效。

2.  **自適應調整 (Adaptive Adjustments):**
    *   響應式設計不是重新做一個網站，而是在原有的桌面版設計基礎上，針對不同斷點進行「微調」。最常見的調整項目包含：
        *   **版面排版：** 例如，電腦版的三欄式版面，在手機上自動堆疊成單欄。
        *   **間距 (Padding/Margin)：** 手機螢幕較小，通常需要縮小區塊的內外距，讓內容更緊湊。
        *   **文字大小 (Font Size)：** 調整標題與內文的字體大小，確保在小螢幕上的易讀性。
        *   **元素可見度 (Visibility)：** 這是一個進階技巧。你可以讓某個複雜的元素（例如大型 Banner）在手機上**隱藏**，改用一個更簡潔的圖片替代，以提升載入速度和使用者體驗。

### Part 1: 進入響應式編輯模式

無論是 Kadence Blocks 還是 Elementor，都提供了非常直覺的響應式編輯模式切換器。

#### **A. Kadence Blocks**

在 Gutenberg (區塊) 編輯器的右上方，你會看到一個裝置圖示。點擊後即可切換預覽模式。

*   **位置：** 編輯頁面的右上角，在「更新」按鈕的左側。
*   **操作：** 點擊「預覽」按鈕旁邊的「桌面」圖示，會出現「平板電腦」和「行動裝置」的選項。

``

#### **B. Elementor**

在 Elementor 編輯器的左下角面板，你會看到一個「響應模式」的圖示。

*   **位置：** 左側工具面板的底部，靠近「預覽變更」的眼睛圖示。
*   **操作：** 點擊「響應模式」圖示後，編輯器頂部會出現一條工具列，讓你可以在「桌面」、「平板」、「手機」之間切換。

``

### Part 2: (實作) 使用 Kadence Blocks 進行響應式調整

讓我們以一個常見的「三欄式服務介紹」區塊為例。

#### **步驟 1: 完成桌面版設計**

首先，請確保你的桌面版設計已經定稿。我們使用「橫列版面」區塊，並在裡面放置三個「區段」區塊來達成三欄式設計。

``

#### **步驟 2: 切換至「平板電腦」模式並調整**

1.  點擊右上角的預覽切換器，選擇「**平板電腦**」。
2.  你會發現三欄可能變得有些擁擠。點選最外層的「**橫列版面**」區塊。
3.  在右側的區塊設定中，找到「**Layout**」設定。在「欄」的設定處，將其從 3 欄改為 2 欄或 1 欄，看哪個佈局更適合。
4.  **Moksa 技巧：** 你會注意到許多設定（如 Padding、Margin）旁邊都有一個**小小的裝置圖示**。這代表此設定可以針對不同裝置分別設定。點擊它，切換到平板圖示，然後調整平板專屬的間距。例如，我們可以減少整個區塊的上下內距 (Padding)，讓版面更緊湊。

``

#### **步驟 3: 切換至「行動裝置」模式並優化**

1.  切換到「**行動裝置**」預覽模式。
2.  通常，Kadence 會自動將多欄式版面堆疊為單欄，這是最適合手機瀏覽的行為。
3.  我們的主要任務是**優化細節**：
    *   **調整間距：** 點選區塊，找到 Padding/Margin 設定，點擊裝置圖示並切換到手機，將左右內距縮小（例如 `15px`），避免內容太貼近螢幕邊緣。
    *   **調整字級：** 點選「進階文字」區塊 (標題)，在右側設定找到「字體大小」，同樣點擊裝置圖示，為手機設定一個較小的、更易讀的字級。

``

#### **步驟 4: 調整元素可見度 (Visibility)**

假設桌面版有一個複雜的背景圖，但在手機上會干擾文字閱讀。

1.  選取該「橫列版面」區塊。
2.  在右側設定面板，切換到「**進階**」分頁，找到「**可見度設定**」。
3.  在這裡，你可以設定「**在手機上隱藏此區塊**」。
4.  接著，你可以新增一個新的、設計更簡潔的「橫列版面」區塊，並在它的「可見度設定」中，設定為「**隱藏於桌面**」和「**隱藏於平板**」。如此一來，這個簡化版的區塊就只會出現在手機上。

### Part 3: (實作) 使用 Elementor 進行響應式調整

同樣以一個三欄式版面為例。

#### **步驟 1: 完成桌面版設計**

在 Elementor 中，我們會新增一個「內部章節」，並設定為三欄。

``

#### **步驟 2: 切換至「平板」模式並調整**

1.  點擊左下角的「**響應模式**」圖示，然後在頂部工具列選擇「**平板**」。
2.  點選你想要調整的「**欄 (Column)**」。
3.  在左側面板的「**版面配置**」分頁中，找到「**欄寬 (%)**」。你會看到這個設定旁邊有裝置圖示。
4.  **Moksa 技巧：** 你可以為每一欄設定在平板上的寬度。例如，你可以將前兩欄設定為 `50%`，第三欄設定為 `100%`，這樣就會變成「兩欄 + 一欄」的佈局。

``

#### **步驟 3: 切換至「手機」模式並優化**

1.  在頂部工具列選擇「**手機**」。
2.  預設情況下，Elementor 會將所有欄寬變為 `100%`，使其自動堆疊。
3.  **優化間距：**
    *   點選整個「**章節 (Section)**」或個別的「**欄 (Column)**」。
    *   前往「**進階**」分頁。
    *   在「**外距 (Margin)**」和「**內距 (Padding)**」設定中，你會看到裝置圖示。請確保你正為「手機」模式進行編輯。
    *   通常我們會取消連結值，並減少手機上的左右內距至 `15px` 左右。
4.  **優化字級：**
    *   點選「**標題 (Heading)**」或「**文字編輯器 (Text Editor)**」小工具。
    *   前往「**樣式**」 > 「**排版**」。
    *   在「**尺寸**」設定旁，點擊裝置圖示，為手機設定合適的字體大小。

``

#### **步驟 4: 調整元素可見度**

1.  選取任何你想要在特定裝置上隱藏的「**章節 (Section)**」、「**欄 (Column)**」或「**小工具 (Widget)**」。
2.  前往左側面板的「**進階**」分頁。
3.  找到「**響應式**」這個區塊並展開它。
4.  在這裡，你可以找到「**在電腦/平板/手機上隱藏**」的開關，根據你的需求進行設定即可。

### 總結與最佳實踐

恭喜你！你已經掌握了 RWD 響應式設計的核心實作技巧。請記住以下幾個最佳實踐：

*   **優先考慮手機 (Mobile First)：** 雖然我們習慣從桌面版開始設計，但在心中要時常思考：「這個設計在手機上看起來會如何？」
*   **保持簡潔：** 手機螢幕寸土寸金，避免放置過多不必要的裝飾性元素。
*   **確保點擊區域：** 手機是透過手指觸控，按鈕、連結的尺寸不能太小，彼此間距也要足夠，避免使用者誤觸。
*   **測試，測試，再測試：** 編輯器內的預覽功能非常方便，但最終還是要**使用真實的手機和平板**去瀏覽你的網站，才能發現最真實的問題。

響應式設計是現代網站開發的必備技能。多加練習，你會發現為不同裝置創造流暢一致的瀏覽體驗，是一件非常有成就感的事！
