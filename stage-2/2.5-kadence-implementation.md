# 02.05 (實作) 使用 Kadence Blocks 實作 Figma 靜態頁面

嗨，歡迎來到這個章節！這是 MoksaWP 課程中非常重要的一環，我們將學習如何將設計師的夢想（Figma 設計稿）轉化為現實（一個功能完整、響應式的 WordPress 頁面）。這項技能是前端開發與網站設計之間最關鍵的橋樑。

在這個實作單元中，我們將一步步帶你走過從分析 Figma 設計稿、提取素材，到使用 **Kadence Blocks** 這個強大的區塊編輯器，像素級地重現設計，並確保它在手機、平板和電腦上都看起來很完美。

---

### 課前準備

在開始之前，請確保你已經準備好以下項目：

1.  **Figma 設計稿的存取權限**：你需要能夠查看設計稿的詳細資訊，包含顏色代碼、字體大小、間距，並能夠匯出圖片等素材。
2.  **一個乾淨的 WordPress 頁面**：前往你的 WordPress 後台，新增一個頁面 (`頁面 > 新增頁面`)，並將其命名，例如「首頁設計實作」。
3.  **耐心與專注**：將設計稿轉換為網頁需要細心，特別是在間距和排版的細節上。

---

### 從設計稿到網頁的思考流程

在我們動手之前，先建立一個專業的思考框架。直接埋頭做通常會導致混亂和返工。

1.  **解構 (Deconstruct)**：將整個頁面設計稿，由上到下，拆解成一個個大的「**區塊 (Section)**」。例如：導覽列下方的主視覺 (Hero Section)、服務介紹區塊、客戶見證區塊、聯絡表單區塊等。
2.  **分析 (Analyze)**：針對每一個區塊，再細部分析它的「**欄位 (Column)**」結構。這是一個兩欄式設計嗎？還是三欄？每個欄位裡面又包含了哪些元素（標題、內文、按鈕、圖片）？
3.  **提取 (Extract)**：從 Figma 中獲取所有必要的設計參數與素材。
    *   **色彩**：記錄下所有會用到的顏色代碼 (例如 `#333333`, `#FF6B6B`)。
    *   **字型**：記錄下標題、內文的字體、字重 (Font Weight)、大小 (Font Size)、行高 (Line Height)。
    *   **間距**：觀察 Section 與 Section 之間的上下間距 (Margin)，以及元素與容器邊界的內距 (Padding)。
    *   **素材**：將所有圖片 (JPG/PNG/WebP)、圖示 (SVG) 都匯出並整理好。
4.  **建構 (Build)**：依照由大到小（Section -> Column -> Elements）的順序，使用 Kadence Blocks 開始堆疊。

---

### 實作步驟

現在，讓我們開始動手吧！

#### 步驟 1: 設定 Kadence 全域色彩與字型

這是專業工作流程的第一步，也是一個重要的 **Moksa 技巧**。與其在每個區塊中手動設定顏色和字體，不如我們先在佈景主題的「自訂」中設定好「**全域調色盤 (Global Palette)**」與「**全域字型 (Global Typography)**」。這樣不僅能確保全站風格統一，未來要修改時也只需要改一個地方。

1.  從 WordPress 後台，前往「**外觀**」>「**自訂**」。
2.  點擊「**色彩與字型**」>「**色彩**」。
3.  你會看到一個全域調色盤。點擊色塊，將你從 Figma 記錄下來的主要顏色、次要顏色、文字顏色等，一一填入。
    `[SCREENSHOT]`
4.  接著，回到上一層，點擊「**字型**」。
5.  根據 Figma 的設計，設定好**基礎內文 (Base Font)** 和**標題 (Headings)** 的字體家族、字重、大小等。
    `[SCREENSHOT]`
6.  完成後，點擊「**發佈**」儲存設定。

#### 步驟 2: 使用 Row Layout 建立第一個區塊 (Hero Section)

`Row Layout` 是 Kadence Blocks 中最核心、最萬能的容器區塊。我們所有的內容都應該放在 `Row Layout` 裡面。

1.  回到你新增的頁面編輯器，點擊 `+` 新增區塊，搜尋並選擇「**Row Layout**」。
2.  Kadence 會詢問你想要的欄位結構。假設我們的 Hero Section 是一個滿版的背景圖，中間有標題和按鈕，我們就選擇「**單欄**」。
    `[SCREENSHOT]`
3.  點選剛剛新增的 `Row Layout` 區塊（你可以透過左下角的「區塊導覽」來精準選取）。在右側的區塊設定面板中，找到「**背景設定 (Background Settings)**」。
4.  上傳你的背景圖片，並設定「**背景尺寸 (Background Size)**」為「**Cover (覆蓋)**」以確保圖片能填滿整個區塊。你也可以加上「**背景覆蓋 (Background Overlay)**」讓文字更清晰。
5.  在「**間距設定 (Spacing Settings)**」中，調整「**內距 (Padding)**」的「上」和「下」，讓這個區塊有足夠的高度。例如，上下都設定為 `120px`。
    `[SCREENSHOT]`

#### 步驟 3: 在區塊中加入內容元素

現在我們的容器已經準備好了，可以開始放入內容。

1.  在 `Row Layout` 區塊中，點擊 `+` 圖示，新增一個「**Advanced Heading**」區塊。
2.  輸入你的主標題文字。在右側的設定面板中，你會發現它的顏色和字體已經自動套用了我們在**步驟 1** 設定的全域樣式。你只需要微調大小或行高即可。
3.  在標題下方，新增一個「**Advanced Text**」區塊來放置副標題或內文。
4.  最後，新增一個「**Advanced Button**」區塊。在設定面板中，你可以輕鬆設定按鈕的文字、連結、顏色、邊框圓角等，直到它看起來跟 Figma 設計稿一模一樣。
    `[SCREENSHOT]`

#### 步驟 4: 響應式設計 (RWD) 調整

這是決定一個網站專業度的關鍵。Kadence Blocks 提供了非常直覺的響應式設計工具。

1.  在頁面編輯器的右下角（或上方），你會看到三個圖示：**電腦、平板、手機**。
    `[SCREENSHOT]`
2.  點擊「**平板 (Tablet)**」圖示，預覽頁面在平板上的樣子。
3.  你會發現，很多設定值旁邊也出現了這三個圖示。這表示這個設定可以針對不同裝置有不同的值。
4.  **Moksa 技巧**：在平板或手機模式下，你可能會覺得標題太大、或是區塊的上下內距太寬。這時，你只需要點擊對應裝置的圖示，然後輸入一個新的數值即可。例如，將標題的字體大小從 `60px`（電腦）調整為 `48px`（平板），再調整為 `36px`（手機）。這個調整只會對該裝置生效，不會影響其他裝置的設定。
    `[SCREENSHOT]`
5.  重複這個流程，切換到「**手機 (Mobile)**」模式，檢查並微調所有元素的間距、字體大小，直到在手機上的排版看起來舒適美觀。

#### 步驟 5: 重複流程，完成整個頁面

現在你已經掌握了建立一個區塊的完整流程。接下來，你只需要：

1.  在現有 `Row Layout`下方新增下一個 `Row Layout` 區塊。
2.  分析 Figma 中下一個區塊的結構（例如，一個三欄的服務介紹區塊）。
3.  選擇對應的欄位佈局，並在每個欄位中填入圖片 (`Advanced Image`)、標題 (`Advanced Heading`) 和文字 (`Advanced Text`)。
4.  不斷重複「**建立容器 -> 填入內容 -> 調整細節 -> RWD 檢查**」這個循環，直到整個頁面都建構完成。

### 總結與最佳實踐

恭喜你！你已經學會了如何將一個靜態的 Figma 設計稿，透過 Kadence Blocks 轉化為一個功能齊全、響應式的動態網頁。

**重點回顧：**

*   **先規劃，再動手**：解構、分析、提取素材是節省時間的關鍵。
*   **善用全域設定**：優先設定全域色彩與字型，保持風格一致性。
*   **`Row Layout` 是你的好朋友**：所有內容都應該由 `Row Layout` 作為最外層的容器開始。
*   **響應式優先**：時刻切換裝置預覽，確保在所有螢幕尺寸上都有最佳體驗。
*   **使用區塊導覽**：當頁面變得複雜時，左下角的「區塊導覽」（或稱為 List View）可以幫助你快速找到並選取任何一個巢狀區塊。

這項技能需要透過不斷練習來精進。試著找一些你喜歡的網站，或是在 Dribbble、Behance 上找一些設計稿，挑戰自己用 Kadence Blocks 把它們實作出來吧！
