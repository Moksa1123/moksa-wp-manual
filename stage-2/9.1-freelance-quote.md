# 09.01 如何報價 (網站類型 vs. 維護合約)

大家好，我是 Moksa 的首席課程導師。在之前的章節中，我們已經掌握了從無到有建立一個功能強大網站的所有技術。然而，將技術轉化為商業價值，最關鍵的一步就是「報價」。

報價不僅僅是提供一個數字，它是你**專業價值的體現**、**與客戶溝通的起點**，更是**保護雙方權益的重要文件**。一個好的報價能讓你接到理想的案子，一個壞的報價則可能讓你陷入無盡的修改、虧損，甚至傷害你的專業品牌。

這堂課，我們將深入探討兩大報價核心：**網站建置專案**與**長期維護合約**。

---

## 1. 核心心態轉變：從「工時思維」到「價值導向」

在報價之前，請先拋棄一個迷思：「這個網站我要花多少小時做？」

初學者常常陷入時薪的陷阱，但客戶在乎的不是你花了多少時間，而是**你為他解決了什麼問題、創造了多少價值**。

一個每年能帶來三百萬營業額的電商網站，和一個只用來展示作品集的形象網站，即便開發時數相近，它們為客戶帶來的商業價值卻有天壤之別。你的報價，必須反映這個價值。

**Moksa 技巧：** 報價的基礎是「信任」。在報價前，務必與客戶進行深度訪談，了解他的**商業目標 (Business Goal)**、**目標客群 (TA)** 以及**痛點 (Pain Point)**。當你的提案是為了解決他的痛點、幫助他達成目標時，價格就只是其次的考量。

---

## 2. 網站建置報價：拆解專案，精準估算

一個完整的網站建置專案，可以拆解為幾個核心部分。我們將以 Moksa 技術棧為基礎，教你如何系統化地進行估價。

### 步驟 1：識別網站類型與核心功能

不同的網站類型，其複雜度與所需工具截然不同。

| 網站類型 | 核心目標 | 常用 Moksa 技術棧 | 報價複雜度 |
| :--- | :--- | :--- | :--- |
| **形象展示網站** | 品牌曝光、資訊傳達、聯絡表單 | Kadence Blocks/Elementor, Rank Math SEO, WP Rocket | ★☆☆☆☆ |
| **部落格/內容網站** | 內容行銷、SEO 流量、廣告收益 | Rank Math SEO, FluentCRM (電子報), ShortPixel | ★★☆☆☆ |
| **電商購物網站** | 線上銷售、訂單管理、會員行銷 | WooCommerce, **綠界金流/LINE Pay**, **RY Tools**, **綠界發票**, FluentCRM, YITH Points, Advanced Coupons | ★★★★☆ |
| **線上課程 (LMS)** | 課程銷售、學員管理、學習進度追蹤 | LearnDash/LifterLMS, WooCommerce (銷售), YITH Membership | ★★★★★ |
| **會員/社群網站** | 會員分級、內容權限、社群互動 | YITH Membership, BuddyBoss, FluentCRM | ★★★★★ |

### 步驟 2：估算工時的黃金公式

一個相對科學的估價公式如下：

> **專案總價 = (各階段預估工時 x 你的時薪) + 外掛/服務授權費 + 風險緩衝 (15-20%)**

**Moksa 技巧：** **絕對不要**只報一個總價！你必須在報價單中詳細列出工作項目，這能體現你的專業度，也能作為未來定義 **工作範疇 (Scope of Work)** 的依據。

### 步驟 3：拆解專案階段與工作項目

以下是一個標準專案的工時拆解範例，你可以根據這個模板來建立自己的報價單。

#### A. 專案前期 (10%)
*   需求訪談與分析
*   網站架構規劃 (Sitemap)
*   使用者流程規劃 (User Flow)
*   撰寫規格文件與報價單

#### B. 設計階段 (15%)
*   首頁與內頁風格設計 (Wireframe & Mockup)
*   **Moksa 技巧：** 即便你是用 Kadence 或 Elementor，也建議先用 Figma 或 Whimsical 等工具確認版面配置，避免進入開發後才大幅修改設計。

#### C. 開發與建置階段 (50%)
這是最核心的部分，務必詳細條列：
*   **WordPress 環境建置：** 主機設定、安裝 WordPress 核心。
*   **基礎外掛安裝與設定：** Kadence Blocks, Rank Math SEO, WP Rocket, UpdraftPlus, Wordfence。
*   **頁面建置：** 根據設計稿，使用 Kadence/Elementor 製作全站頁面 (首頁, 關於我們, 聯絡我們...等)。
*   **核心功能建置 (依網站類型而定)：**
    *   **電商功能：**
        *   安裝與設定 WooCommerce。
        *   設定 **綠界 ECPay (信用卡/ATM)** 與 **LINE Pay** 金流。
        *   設定 **RY Tools** (超商取貨、黑貓宅配)。
        *   設定 **綠界電子發票** (串接與自動開立)。
        *   設定稅金、運費、商品分類與標籤。
    *   **行銷功能：**
        *   設定 **FluentCRM** 自動化流程 (如：棄單召回、新戶歡迎信)。
        *   設定 **Advanced Coupons** (如：BOGO 買一送一、URL 優惠券)。
        *   設定 **YITH Points and Rewards** 點數回饋機制。
    *   **會員/課程功能：**
        *   設定 **YITH Membership** 或 **LearnDash** 的方案、權限與課程結構。
*   **內容上稿：** 協助客戶上架 N 篇產品/文章/課程 (務必在報價單中**明確定義數量**)。
*   **SEO 基礎設定：** 使用 Rank Math 設定 Title/Meta、Sitemap 提交、連接 Google Search Console。

#### D. 測試與上線 (15%)
*   跨瀏覽器與響應式 (RWD) 測試。
*   功能測試 (表單、購物車、金流、會員登入)。
*   壓力與速度測試。
*   網站部署上線。
*   **Moksa 技巧：** 務必保留完整的測試時間，尤其金流串接，一定要用測試卡號與測試模式完整跑過一次購物流程。

#### E. 教育訓練與結案 (10%)
*   後台操作手冊撰寫。
*   一對一線上教育訓練 (e.g., 1.5 小時)。
*   交付網站所有權限與資料。

## 3. 網站維護合約：創造穩定的現金流

專案結束不該是服務的終點。一個健康的網站需要持續的維護，這對你而言是**穩定的經常性收入 (Recurring Revenue)**，對客戶而言則是**網站安全的保障**。

### 為什麼客戶需要維護合約？
*   **安全：** WordPress 是駭客攻擊的熱門目標，定期更新與掃描至關重要。
*   **效能：** 網站會隨著時間累積垃圾資料，需要定期優化。
*   **穩定：** 外掛與核心的更新可能產生衝突，需要專業人員處理。
*   **安心：** 客戶可以專注於他的本業，將網站技術問題交給你。

### 維護合約報價方案（範例）

建議提供至少兩種方案，讓客戶依據需求選擇。

| 服務項目 | **基礎維護方案** (適合形象網站) | **進階商務方案** (適合電商/會員站) |
| :--- | :--- | :--- |
| **報價參考 (月費)** | NT$ 3,000 - 5,000 | NT$ 8,000 - 15,000+ |
| **網站備份 (UpdraftPlus)** | 每週一次 (異地) | 每日一次 (異地) |
| **安全掃描 (Wordfence)** | 每週一次 | 每日一次 |
| **外掛/主題/核心更新** | 每月一次 (在 Staging 環境測試) | 每週一次 (在 Staging 環境測試) |
| **資料庫優化** | 每月一次 | 每週一次 |
| **效能監控 (WP Rocket)** | 每月一次 | 每週一次 |
| **技術支援時數** | 1 小時/月 (Email/LINE) | 3 小時/月 (Email/LINE/電話) |
| **網站報告** | 每月一次 (摘要報告) | 每月一次 (詳細數據與建議報告) |
| **緊急狀況處理** | 上班時間 | 優先處理 / 假日支援 |

``
*(此處可放一個精美的維護方案比較表格圖片)*

**Moksa 技巧：** 在專案結案時，主動提出維護合約的必要性。可以提供「**首年簽訂維護合約享 9 折優惠**」作為誘因，提高簽約率。同時，明確定義合約中「技術支援時數」的**使用範圍**與**超出時數的收費標準**，避免爭議。

## 總結

一份專業的報價，是你展現實力、贏得客戶信任的第一步。

1.  **心態：** 永遠以「**價值**」為核心，而非「工時」。
2.  **建置報價：** 將專案**徹底拆解**，讓客戶清楚知道每一分錢花在哪裡。使用我們的 Moksa 技術棧作為你的項目檢查清單，確保沒有遺漏。
3.  **維護合約：** 這是你事業的**護城河**，為你帶來穩定的收入，也為客戶提供長期的保障。
4.  **透明化：** 在報價單中，永遠要包含**工作範疇 (Scope of Work)**、**付款條件 (Payment Term)** 與**時程規劃 (Timeline)**，這會為你省下大量的溝通成本與潛在糾紛。
