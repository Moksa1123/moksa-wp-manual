# 01.03 頁面編輯器 (Page Builder) 總覽 (Elementor vs Kadence)

你好，我是 Moksa 的首席課程導師。歡迎來到這個關鍵的單元。

在 WordPress 的世界裡，「頁面編輯器 (Page Builder)」是你將腦中藍圖化為實際網頁的畫筆與工具箱。選擇哪一款編輯器，將會深深地影響你網站的設計彈性、載入速度、長期維護性，甚至是未來的擴充能力。

在這個章節，我們將深入探討目前市場上兩大主流的頁面編輯器：傳統的王者 **Elementor**，以及基於 WordPress 核心區塊編輯器 (Gutenberg) 的新世代領航者 **Kadence Blocks**。理解它們之間的差異與哲學，是你成為專業網站建置者的第一步。

---

### 什麼是頁面編輯器？

簡單來說，頁面編輯器是一種 WordPress 外掛，它提供了一個**視覺化、拖拉式的介面**，讓使用者不需要撰寫任何程式碼，就能夠建立複雜且美觀的頁面佈局。你可以像堆疊樂高積木一樣，將各種元素（如標題、圖片、按鈕、欄位）拖曳到頁面上，並即時看到成果。

### 1. 傳統頁面編輯器的王者 - Elementor

Elementor 是目前全球擁有最多使用者的頁面編輯器，它的出現徹底改變了 WordPress 的設計生態。它以其強大的功能和極高的自由度聞名。

#### Elementor 的運作方式

Elementor 創造了一個**獨立於 WordPress 預設編輯器之外的編輯環境**。當你點擊「使用 Elementor 編輯」時，你會進入一個全新的全螢幕介面，與 WordPress 後台完全脫鉤。

``
*（示意圖：Elementor 的編輯介面，左側為各種 Widgets，右側為頁面畫布）*

#### 優點

*   **功能極其豐富：** 幾乎所有你能想到的網頁效果和功能，Elementor 都有對應的元件 (Widget) 可以實現。
*   **龐大的生態系：** 擁有海量的第三方附加元件 (Add-ons) 和現成範本 (Templates)，擴充性非常強。
*   **成熟的社群：** 使用者多，代表網路上的教學資源、討論區和解決方案也相對豐富。
*   **絕對的視覺控制：** 對於像素級的精準排版、絕對定位等，提供了非常細緻的控制選項。

#### 缺點

*   **效能問題：** 這是 Elementor 最為人詬病的一點。為了實現高度客製化，它會產生大量巢狀的 `<div>` 標籤，俗稱「Div-ception」。這會導致 HTML 結構臃腫，影響網站載入速度 (PageSpeed Scores)。
*   **供應商鎖定 (Vendor Lock-in)：** **這是一個嚴重的商業風險**。如果你用 Elementor 製作了內容，未來某天決定停用它，你的頁面內容將會崩潰，只留下一堆無法解讀的「短代碼 (Shortcodes)」。這意味著你的網站內容被這個外掛「綁架」了。
*   **較耗資源：** 在後台編輯時，尤其是在複雜的頁面上，Elementor 的編輯器可能會感覺到卡頓或延遲。

---

### 2. 新世代區塊編輯器的領航者 - Kadence Blocks

相對於 Elementor 取代原有編輯器的做法，Kadence Blocks 走了一條完全不同的路。它**不是取代，而是「增強」** WordPress 內建的區塊編輯器 (Gutenberg)。

#### Kadence Blocks 的運作方式

Kadence Blocks 直接在 WordPress 原生的編輯介面中運作。它提供了一系列功能更強大、設定更豐富的「區塊」，讓你用原生、高效能的方式，達到媲美傳統頁面編輯器的設計效果。

``
*（示意圖：WordPress 原生區塊編輯器介面，但右側設定欄顯示著 Kadence Blocks 提供的進階選項）*

#### 優點

*   **頂級的網站效能：** **這是 Moksa 選擇它的最主要原因**。Kadence Blocks 產生的程式碼非常乾淨、輕量，幾乎沒有多餘的程式碼，這讓它在 Google PageSpeed Insights 等效能測試中表現極佳。
*   **與 WordPress 核心整合：** 它是順應著 WordPress 官方發展方向的工具。隨著區塊編輯器越來越強大，Kadence Blocks 的未來發展潛力無窮。
*   **無供應商鎖定：** 如果你停用 Kadence Blocks，內容不會消失。雖然一些進階的排版樣式會遺失，但你的文字、圖片等核心內容會平滑地轉換為 WordPress 的原生區塊，網站不會因此崩潰。
*   **直覺且快速的編輯體驗：** 由於是原生介面，後台的載入和反應速度非常快，提供了流暢的編輯流程。

#### 缺點

*   **生態系較小：** 雖然正在快速成長，但第三方的附加元件和範本數量目前仍不及 Elementor。
*   **極端複雜的設計：** 對於一些需要絕對定位、重疊圖層等非常規的設計，Elementor 可能會提供更直覺的操作方式。

---

### Elementor vs. Kadence Blocks - 核心差異比較

| 功能 | Elementor | Kadence Blocks |
| :--- | :--- | :--- |
| **編輯哲學** | **取代** WordPress 編輯器，提供獨立介面 | **增強** WordPress 核心區塊編輯器 |
| **網站效能** | 較重，程式碼較臃腫，可能影響速度 | **極致輕量**，程式碼乾淨，載入速度快 |
| **供應商鎖定** | **高風險**，停用後內容會變成短代碼 | **低風險**，內容可降級為原生區塊 |
| **學習曲線** | 容易上手，但精通所有設定需要時間 | 與 WordPress 核心體驗一致，學習曲線平滑 |
| **生態系** | 極其龐大且成熟 | 正在快速成長，品質精良 |
| **未來趨勢** | 獨立發展，與 WordPress 核心有隔閡 | 與 WordPress 核心同步發展，前景看好 |

### Moksa 的觀點與專業建議

在 Moksa，我們經手過數百個網站的開發與維護專案。基於長期的實務經驗，我們得出一個非常明確的結論。

**Moksa 技巧：我們的標準技術棧明確推薦使用 Kadence Blocks 作為主要的頁面編輯器。**

理由如下：

1.  **效能是電商的生命線：** 對於一個商業網站，尤其是 WooCommerce 電商網站，網站速度直接影響轉換率與 SEO 排名 (Rank Math)。Kadence Blocks 在效能上的巨大優勢，是我們無法忽視的。一個快速的網站能帶來更好的使用者體驗，並最終轉化為更高的營收。
2.  **擁抱未來，而非對抗趨勢：** WordPress 官方已經將全部的開發重心都放在區塊編輯器上。選擇 Kadence Blocks 意味著你的網站技術是與 WordPress 的未來發展方向一致的，這能確保長期的穩定性與相容性。
3.  **避免技術負債與風險：** 「供應商鎖定」是專業開發者極力避免的陷阱。我們希望客戶的網站資產是自由、可控的，而不是被單一外掛綁架。Kadence Blocks 讓我們擁有這種自由。
4.  **簡潔就是力量：** 更乾淨的程式碼意味著更少的潛在衝突、更容易的除錯，以及更低的長期維護成本。

雖然 Elementor 是一款強大的工具，但我們認為它更適合快速製作 Landing Page 或非核心的行銷頁面。對於一個需要長期經營、重視效能與穩定性的商業網站來說，Kadence Blocks 無疑是更專業、更明智的選擇。

在接下來的課程中，我們所有的實作教學，都將會以 **Kadence Theme** 與 **Kadence Blocks** 為基礎來進行。
