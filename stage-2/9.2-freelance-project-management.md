# 09.02 (實務) 專案管理 (SOW、時程、驗收)

你好，我是 Moksa 的首席課程導師。

歡迎來到網站設計與進階客製化的實務管理章節。在前面的課程中，我們已經掌握了強大的技術工具，但**真正決定一個專案成敗、能否獲利、客戶是否滿意的關鍵，往往不在於程式碼，而在於「管理」**。

很多開發者都曾遇過「需求不斷追加」、「客戶說的跟我想的不一樣」、「專案延遲」、「尾款收不到」等惡夢。這堂課的目標，就是帶你建立一套專業的專案管理流程，從源頭避免這些問題。我們將專注於三個最重要的環節：**SOW (專案範疇說明書)**、**時程規劃**，以及**驗收結案**。

## 一、專案範疇說明書 (SOW - Statement of Work)：一切的基礎

SOW 是一份專案的「合約」與「說明書」，它**用白紙黑字明確定義了買賣雙方（客戶與你）的權利與義務**。一份清晰的 SOW 可以防止「範疇蠕變 (Scope Creep)」，也就是客戶不斷追加原先沒提到的新功能。它更是專案時程、報價與最終驗收的唯一依據。

### 為什麼 SOW 如此重要？

*   **建立共識：** 確保你和客戶對專案的目標、功能、交付成果有完全相同的理解。
*   **保護雙方：** 對開發者而言，它界定了工作範圍，超出範圍的功能可以另外報價；對客戶而言，它確保了開發者會交付所有承諾的功能。
*   **報價依據：** 你的專案報價，完全是基於 SOW 中所列出的項目。沒有 SOW，報價就只是憑感覺喊價。

### 一份專業的 SOW 應該包含什麼？

1.  **專案目標 (Project Goals):**
    *   客戶為什麼要做這個網站？是為了提升品牌形象、增加線上銷售、還是收集客戶名單？
    *   範例：「本專案旨在建立一個具備線上購物、會員點數、課程販售功能的美妝品牌官方網站，期望在網站上線後三個月內，線上業績能佔總業績的 20%。」

2.  **工作範疇 (Scope of Work):**
    *   這是 SOW 的核心，必須鉅細靡遺。你**必須明確列出「包含」與「不包含」的項目**。

    ``

    *   **Moksa 技巧：** 將我們的技術棧具體化，可以讓範疇更清晰，避免模糊空間。

    > **範例：包含的項目 (In-Scope)**
    >
    > *   **網站架構：** 基於 WordPress 與 WooCommerce。
    > *   **頁面設計：** 使用 Kadence Blocks Pro，完成首頁、關於我們、商品列表頁、商品內頁、購物車、結帳頁，共 6 個頁面的主要版型設計。
    > *   **金流串接：**
    >     *   綠界 ECPay 信用卡 (一次付清)。
    >     *   綠界 ECPay 虛擬 ATM。
    >     *   LINE Pay。
    > *   **物流設定：**
    >     *   使用 RY Tools 設定 7-11 與全家超商取貨 (C2C)。
    >     *   設定宅配 (固定運費)。
    > *   **發票設定：** 串接綠界 ECPay 電子發票，訂單完成後自動開立。
    > *   **行銷功能：**
    >     *   使用 FluentCRM 設定 3 封棄單召回自動化 Email。
    >     *   使用 YITH Points and Rewards 設定「消費 100 元回饋 1 點，1 點折抵 1 元」的點數規則。
    > *   **SEO 基礎設定：** 使用 Rank Math SEO 完成全站 SEO 標題與描述格式設定，提交 Sitemap 至 Google Search Console。
    > *   **效能優化：** 安裝與設定 WP Rocket, ShortPixel 進行基礎效能調校。

    > **範例：不包含的項目 (Out-of-Scope)**
    >
    > *   **所有商品** 的上架作業 (僅上架 5 項測試商品)。
    > *   網站 Logo、Banner、商品圖片等**視覺素材設計** (由客戶提供)。
    > *   網站所有頁面的**文案撰寫** (由客戶提供)。
    > *   串接 ERP、CRM 等第三方系統。
    > *   Facebook/Google 廣告投放與設定。
    > *   網站上線後的長期維護與主機代管 (可另簽維護合約)。

3.  **交付項目 (Deliverables):**
    *   專案完成後，你會「交給」客戶什麼東西。
    *   範例：完整的 WordPress 網站原始碼、網站後台最高管理員權限、UpdraftPlus 首次完整網站備份檔、專案相關文件 (如 SOW、驗收清單)。

4.  **假設與前提 (Assumptions):**
    *   專案能順利進行所依賴的條件。
    *   範例：客戶需在收到待確認項目後的 3 個工作天內回覆；客戶需在專案啟動前提供所有必要的圖文資料。

## 二、時程規劃 (Timeline)：將承諾化為行動

有了 SOW，我們就能將巨大的專案拆解成可管理的小任務，並安排執行的時間。一個好的時程表不僅能讓客戶安心，也能讓你有效地管理自己的工作。

### **步驟 1：拆解 SOW 任務**

將 SOW 中「包含的項目」逐一列出，並拆解成更細的執行任務。

*   **範例：**
    *   「金流串接」可以拆解成：
        1.  協助客戶申請綠界/LINE Pay 帳號。
        2.  安裝綠界/LINE Pay 外掛。
        3.  取得 API Key 並填入設定。
        4.  進行測試刷卡。
        5.  設定退款 API。

### **步驟 2：估算工時並加入緩衝**

為每一個小任務估算需要的「專注工作時間 (小時/天)」。

*   **Moksa 技巧：** **永遠要加入緩衝時間！** 你的估算時間 `x 1.3` 或 `x 1.5` 是一個比較安全的做法。緩衝時間是用來應對：
    *   **技術問題：** 外掛衝突、API 串接失敗等。
    *   **溝通成本：** 與客戶來回確認需求的時間。
    *   **修改時間：** 客戶提出修改意見後，你需要時間來調整。

### **步驟 3：安排里程碑 (Milestones)**

將相關的任務組合成一個「里程碑」，並設定明確的交付日期。這有助於分階段向客戶展示進度並收款。

*   **範例時程規劃 (甘特圖):**

``

| 里程碑 | 內容 | 預計完成日期 |
| :--- | :--- | :--- |
| **第一階段 (Week 1-2)** | 需求訪談、SOW 確認、網站架構規劃、主機環境建置 | 2023-10-15 |
| **第二階段 (Week 3-4)** | 首頁與主要頁面版型設計 (UI/UX)、客戶初版確認 | 2023-10-30 |
| **第三階段 (Week 5-6)** | WooCommerce 功能建置 (金流、物流、發票、行銷外掛) | 2023-11-15 |
| **第四階段 (Week 7)** | 內部測試、資料匯入、效能與 SEO 基礎優化 | 2023-11-22 |
| **第五階段 (Week 8)** | 客戶驗收 (UAT)、教育訓練、正式上線、交付尾款 | 2023-11-30 |

## 三、驗收與結案 (Acceptance)：完美交付的最後一哩路

驗收是專案的終點線。它不是一個口頭上的「OK，沒問題了」，而是一個**正式、有憑有據的確認過程**。驗收的唯一標準，就是最一開始雙方簽署的 **SOW**。

### **步驟 1：建立驗收清單 (Acceptance Checklist)**

在專案後期，你需要根據 SOW 的「工作範疇」製作一份詳細的驗收清單。這份清單將是你和客戶逐項確認的依據。

``

### **步驟 2：內部測試 (Alpha Test)**

在交付給客戶前，你或你的團隊必須先按照驗收清單，把所有功能都徹底測試一遍。

### **步驟 3：客戶驗收 (UAT - User Acceptance Test)**

提供網站的測試環境與驗收清單給客戶，請他們在約定的時間內（例如 3-5 個工作天）完成測試並回饋問題。

*   **重點：** 客戶提出的問題，如果是 SOW **範圍內**的功能 bug，你有義務修復。如果客戶提出的是 SOW **範圍外**的新增需求，你應該友善但堅定地告知對方：「這個功能很棒！不過它不在我們這次的合約範圍內，我們可以等這個專案結案後，另外立一個新的專案來為您處理。」

### **步驟 4：正式簽署結案**

當客戶在驗收清單上完成所有項目的勾選與確認後，可以請對方簽署一份「專案結案確認書」，代表專案正式完成，並依合約支付尾款。

*   **Moksa 技巧：** 提供一份基於我們技術棧的驗收清單範本，能讓你的驗收過程更專業、更順暢。

> **驗收清單範例 (部分)**
>
> | 分類 | 驗收項目 | 測試步驟 | 測試結果 |
> | :--- | :--- | :--- | :--- |
| **金流** | **綠界信用卡** | 1. 將商品加入購物車並結帳。 <br> 2. 選擇「信用卡」付款。 <br> 3. 使用綠界測試卡號完成 1 元訂單。 <br> 4. 確認後台訂單狀態為「處理中」。 | ☐ 通過 |
| **物流** | **RY Tools 超商取貨** | 1. 結帳頁面選擇「超商取貨」。 <br> 2. 點擊「選擇門市」，確認電子地圖正常跳出。 <br> 3. 選擇一間門市後，確認資訊有帶回結帳頁面。 | ☐ 通過 |
| **行銷** | **FluentCRM 棄單召回** | 1. 會員登入狀態下，將商品加入購物車。 <br> 2. 填寫結帳資料但不付款，關閉頁面。 <br> 3. 等待 30 分鐘，確認是否有收到第一封棄單提醒信。 | ☐ 通過 |
| **發票** | **綠界電子發票** | 1. 完成一筆測試訂單。 <br> 2. 將後台訂單狀態手動改為「已完成」。 <br> 3. 確認綠界發票後台有同步成立該筆發票。 | ☐ 通過 |

## 總結

掌握「SOW、時程、驗收」這三大支柱，你就從一個單純的「網站開發者」晉升為一位專業的「**網站專案顧問**」。這個流程看似繁瑣，但它能幫你省下無數溝通成本、避免紛爭，最終讓你交付出令客戶滿意、自己也有合理利潤的專案。
