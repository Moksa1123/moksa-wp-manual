# 01.02 本地端開發環境建置 (LocalWP)

嗨，歡迎來到 MoksaWP 網站設計與進階客製化課程！在我們開始進行任何網站的設計、修改程式碼或測試新功能之前，首要之務就是建立一個安全、獨立且高效的「**本地端開發環境**」。這就像是建築師的設計草圖、廚師的備料區，一個讓我們可以盡情揮灑創意而不用擔心搞砸線上網站的沙盒空間。

在本章節，我們將會介紹並帶你一步步安裝 Moksa 團隊一致推薦的本地端開發工具：**LocalWP**。

---

### 為什麼需要本地端開發環境？

想像一下，如果你直接在正在營運的線上網站（我們稱之為「正式站」或 "Production Site"）上修改設計、安裝新外掛或調整程式碼，可能會發生什麼事？

*   **網站可能崩潰：** 一個小小的程式錯誤或外掛衝突，就可能導致整個網站出現白畫面 (White Screen of Death)，讓所有訪客都無法瀏覽。
*   **訪客體驗不佳：** 你在調整顏色、版面配置的過程中，訪客可能會看到一個「施工中」的混亂網站。
*   **資料遺失風險：** 錯誤的操作可能會損壞資料庫，導致訂單、會員資料等重要資訊遺失。
*   **開發效率低落：** 每次修改都要透過 FTP 上傳檔案，等待伺服器反應，速度非常慢。

**本地端開發環境 (Local Environment)** 就是在你自己的電腦上（筆電或桌機）建立一個完整的、與真實主機環境極為相似的 WordPress 網站。在這裡所做的任何修改，都只會影響你電腦裡的這個網站副本，完全不會動到線上的正式站。

**Moksa 技巧：** 專業的開發流程一定是「**本地開發 -> 測試站預覽 -> 正式站上線**」。建立本地端環境，是踏出專業開發者的第一步。

### 我們為什麼推薦 LocalWP？

市面上有許多本地端開發工具（如 XAMPP, MAMP），但 LocalWP (前身為 Local by Flywheel) 因其專為 WordPress 設計的特性而脫穎而出：

*   **一鍵安裝 WordPress：** 無需手動設定資料庫、下載 WordPress 核心檔案，點幾下滑鼠就能建立一個全新的網站。
*   **獨立站點環境：** 每個網站都有自己獨立的 PHP 版本、伺服器軟體 (Nginx/Apache) 和 MySQL 資料庫，方便模擬不同的主機環境。
*   **實用內建工具：**
    *   **MailHog：** 攔截所有 WordPress 寄出的信件（如註冊通知、訂單信），讓你直接在 LocalWP 裡查看，無需設定真實的 SMTP。
    *   **Live Links：** 產生一個臨時的公開網址，讓你輕鬆將本地端網站分享給客戶或同事預覽，無需將網站上傳到主機。
    *   **資料庫管理：** 內建 Adminer，方便直接存取和管理資料庫。
*   **跨平台支援：** 無論你使用 Windows、macOS 還是 Linux，都能有一致的體驗。

### (實作) 安裝 LocalWP 與建立你的第一個本地網站

接下來，讓我們一起動手安裝 LocalWP 並建立你的第一個練習用網站。

#### 步驟 1：下載 LocalWP

首先，請前往 LocalWP 的官方網站。

*   **官方網址：** https://localwp.com/

``
<-- 顯示 LocalWP 官網首頁的截圖 -->
``

點擊畫面上的「Download」按鈕，網站會請你選擇作業系統 (macOS, Windows, Linux) 並填寫一些基本資料，完成後即可開始下載安裝檔。

#### 步驟 2：安裝 LocalWP 應用程式

下載完成後，請執行安裝檔。安裝過程非常直觀，基本上就是不斷點擊「下一步」或「同意」。

*   **Windows 使用者：** 可能會跳出數次「使用者帳戶控制」或「Windows Defender 防火牆」的權限請求視窗，請務必點選「**允許存取**」，讓 LocalWP 可以正常設定你的網路和主機環境。
*   **macOS 使用者：** 將 Local 圖示拖曳到「應用程式」資料夾即可。第一次開啟時可能會需要你輸入管理者密碼。

``
<-- 顯示 LocalWP 安裝過程中的權限請求視窗截圖 -->
``

#### 步驟 3：建立你的第一個本地端網站

安裝完成並開啟 LocalWP 後，你會看到一個簡潔的介面。讓我們來建立網站。

1.  點擊左下角的「**+**」按鈕，或是畫面中間的「**Create a new site**」。

    ``
    <-- 顯示 LocalWP 主介面，並標示出「+」按鈕 -->
    ``

2.  選擇「**Create a new site**」，然後點擊「Continue」。

3.  輸入你的網站名稱，例如 `moksa-practice`。這個名稱會成為你本地端網址的一部分。
    *   **Moksa 技巧：** 點開「**Advanced options**」，你可以自訂本地端網域 (Local site domain) 和網站檔案的路徑 (Local site path)。通常維持預設即可。

    ``
    <-- 顯示輸入網站名稱的畫面，並框選 Advanced options -->
    ``

4.  選擇你的環境 (Environment)。這裡有兩個選項：
    *   **Preferred (推薦)：** 使用 LocalWP 推薦的最新穩定版 PHP、Nginx 和 MySQL。對於初學者或絕大多數情況，**請直接選擇這個選項**。
    *   **Custom：** 可以自訂 PHP 版本、Web Server (Nginx/Apache) 和資料庫版本 (MySQL/MariaDB)。當你需要完全複製線上主機的特定環境時，才會使用此選項。

    ``
    <-- 顯示選擇環境 (Preferred / Custom) 的畫面 -->
    ``

5.  設定 WordPress 管理者帳號。
    *   **WordPress Username:** **請勿使用** `admin`，養成良好資安習慣。可以設定為你常用的英文名或代號。
    *   **WordPress Password:** 設定一組夠強度的密碼。
    *   **WordPress Email:** 預設會填入你註冊 LocalWP 的信箱，這會是網站管理員的信箱。
    *   在「Advanced options」中，你可以選擇這是否為一個「多站點 (Multisite)」網站，目前請保持「**No**」。

    ``
    <-- 顯示設定 WordPress 管理者帳號密碼的畫面 -->
    ``

6.  點擊「**Add Site**」。LocalWP 會開始自動下載 WordPress、設定資料庫、並完成所有伺服器配置。這個過程需要幾分鐘，請耐心等候。過程中如果再次跳出權限請求視窗，請一律選擇「允許」或輸入你的電腦密碼。

#### 步驟 4：啟動並瀏覽你的新網站

當網站建立完成後，你會被帶到該網站的管理儀表板。

``
<-- 顯示單一網站的管理儀表板 (Overview) 截圖 -->
``

在這個畫面，你可以看到幾個重要的資訊和按鈕：

*   **網站狀態：** 右上角會有一個綠色的「Running」或灰色的「Stopped」狀態燈。你可以點擊「**Stop site**」來關閉網站（以節省電腦資源），或在關閉後點擊「**Start site**」來啟動它。
*   **Open Site：** 點擊後會用你的預設瀏覽器開啟網站前台。
*   **WP Admin：** 點擊後會直接開啟 WordPress 後台登入頁面。請用你剛剛設定的帳號密碼登入。
*   **SSL：** 預設是「Not Trusted」。點擊旁邊的「**Trust**」按鈕，並授權後，你的本地端網址 (`https://moksa-practice.local`) 就能啟用 HTTPS，避免瀏覽器顯示不安全警告。
*   **Web Server / PHP / Database：** 顯示目前網站使用的環境版本。

**恭喜！** 你已經成功在自己的電腦上，建立了一個功能完整的 WordPress 網站。現在，你可以在這個安全的環境中，隨心所欲地進行接下來課程中的所有練習，包含安裝佈景主題、外掛、甚至是修改程式碼，都不用擔心會影響任何線上的真實網站。

在下一個章節，我們將會介紹如何將一個線上的網站完整複製到你的 LocalWP 環境中，方便你對現有網站進行維護與開發。
