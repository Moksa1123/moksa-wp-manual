# 03.04 (實務) RWD 實務除錯 (Media Queries)

嗨，歡迎來到這個實務章節！在現代的網頁設計中，RWD (Responsive Web Design，響應式網頁設計) 已經不是一個選項，而是一個**基本要求**。使用者會透過各種尺寸的裝置瀏覽你的網站，從大型桌面螢幕到小尺寸的手機。然而，有時候在桌面版看起來完美的設計，在手機上卻可能發生排版錯亂、文字過大或元素重疊的問題。

這堂課的目標，就是教你一套**專業的除錯流程**，學習如何使用瀏覽器的「開發者工具」精準找出 RWD 問題，並利用 CSS Media Queries 來修正它。這項技能將讓你對網站的掌控力提升到一個新的層次。

---

## 什麼是 Media Queries？

在我們開始除錯之前，先來理解我們的核心武器：**Media Queries**。

簡單來說，Media Queries 是 CSS 的一部分，它像是一個 `if` 條件判斷式。你可以告訴瀏覽器：「**如果**符合某些條件（例如：螢幕寬度小於 768px），**就**套用這組特別的 CSS 樣式」。

這讓我們可以為不同的螢幕尺寸或裝置特性，定義不同的排版、字體大小或顏色，進而實現完美的 RWD 效果。

---

## RWD 除錯的標準流程 (SOP)

遇到 RWD 問題時，不要慌張。跟著 Moksa 的標準流程，你可以有條不紊地解決絕大多數的排版問題。我們將以 Google Chrome 瀏覽器的開發者工具為例，但 Firefox、Edge 等現代瀏覽器也有一樣的功能。

### 步驟 1：開啟瀏覽器開發者工具並進入「響應式設計模式」

首先，在你想要除錯的頁面，點擊滑鼠右鍵，選擇「**檢查 (Inspect)**」，或直接按下鍵盤的 `F12` 快速鍵。

接著，點擊開發者工具左上角的「**切換裝置工具列 (Toggle device toolbar)**」圖示（看起來像一台平板和手機）。

``

你的網頁畫面會立刻縮小，並進入一個可以模擬各種裝置尺寸的模式。你可以從上方的下拉選單選擇預設的裝置（如 iPhone 12 Pro），或手動拖曳邊框來改變視窗寬度。**這一步是所有 RWD 除錯的起點**。

### 步驟 2：鎖定並檢查有問題的 HTML 元素

在響應式模式下，滑動頁面，找到排版出問題的地方。

接著，點擊開發者工具最左上角的「**選取元素 (Select an element)**」圖示（一個鼠標在方框內的圖示）。

``

啟用後，將你的滑鼠移動到網頁上有問題的元素上，例如一個跑版的標題或圖片，然後點擊它。此時，開發者工具的「Elements (元素)」面板會自動跳到該元素的 HTML 程式碼位置，同時「Styles (樣式)」面板會顯示所有套用在該元素上的 CSS 規則。

### 步驟 3：分析並在瀏覽器中即時測試修改

這是最關鍵的一步。在右側的「Styles (樣式)」面板，你可以看到所有影響這個元素的 CSS 樣式，包含 `font-size`, `padding`, `margin`, `width` 等等。

``

你可以直接在這個面板裡進行**即時修改**來測試效果：
*   **取消勾選屬性：** 點擊 CSS 屬性（如 `font-size: 48px;`）前面的小勾勾，可以暫時停用這個樣式，看看排版會有什麼變化。
*   **修改屬性值：** 直接點擊屬性值（如 `48px`），並將其修改為你想要的值（如 `28px`），你會看到網頁畫面**立即更新**。

**Moksa 技巧：** 這個階段的所有修改都**只是暫時的**，重新整理頁面後就會消失。它的目的是讓我們用最快的方式「試出」正確的 CSS 數值，而不用一直儲存檔案再重新整理。

### 步驟 4：撰寫正確的 Media Query 語法

當你在步驟 3 找到正確的 CSS 修正數值後，就該把它寫成正式的 Media Query 了。

最常見的語法是 `max-width`，意思是「當螢幕寬度**小於或等於**指定值時」套用。

**基本語法結構：**

```css
@media (max-width: 768px) {
  /* 在這裡放入你要修改的 CSS 規則 */
  .your-css-selector {
    property: value;
  }
}
```
*   `@media (max-width: 768px)`: 這是條件。`768px` 是一個常用的**斷點 (Breakpoint)**，通常用來區分桌面與平板裝置。
*   `.your-css-selector`: 這是你在步驟 2 鎖定的元素選擇器（可能是 class 或 id）。
*   `property: value;`: 這是你在步驟 3 測試出來的正確樣式。

## 在 WordPress 中實作修正

知道如何修正後，我們要把這段 CSS 程式碼放到網站上。在 Moksa 的技術棧中，我們有兩種主要的方法。

### 方法一：優先使用頁面編輯器的內建響應式設定 (Moksa 推薦)

在你撰寫任何自訂 CSS 之前，**永遠先檢查你的頁面編輯器 (Kadence Blocks 或 Elementor) 是否有內建的響應式設定**。

這兩個編輯器在許多設定（如間距 Padding/Margin、字體大小、對齊方式、欄位寬度等）旁邊，都會有桌面、平板、手機的圖示。

``

你可以點擊這些圖示，分別為不同的裝置設定不同的數值。例如，你可以設定一個標題在桌面上字體大小為 `50px`，但在手機上是 `30px`。

**這是最快、最直觀且最不需要寫程式碼的方式，請務必優先使用。**

### 方法二：當內建設定不足時，使用自訂 CSS

如果頁面編輯器的設定無法滿足你的需求，或是你需要修改的元素不是由編輯器控制的（例如選單或頁尾），這時就輪到我們在步驟 4 寫好的 Media Query 上場了。

**步驟 1：** 從後台儀表板，前往「**外觀**」>「**自訂**」。

**步驟 2：** 進入 WordPress 的自訂器介面後，點選左側選單最下方的「**附加的 CSS (Additional CSS)**」。

``

**步驟 3：** 將你剛剛寫好的 Media Query 程式碼完整貼到這個欄位中。

**實例：** 假設我們發現一個 class 為 `.page-hero-title` 的標題在手機上字體太大，我們在瀏覽器測試後發現 `28px` 是最適合的大小。

```css
/* --- 手機版英雄區域標題樣式調整 --- */

/* 當螢幕寬度小於等於 768px (平板與手機) 時套用 */
@media (max-width: 768px) {
  .page-hero-title {
    font-size: 28px !important; /* 強制覆寫字體大小 */
    line-height: 1.4; /* 調整行高，增加易讀性 */
  }
}
```

**步驟 4：** 點擊上方的「**發佈**」按鈕儲存。

**Moksa 技巧：** 為什麼範例中要加上 `!important`？因為 WordPress 的主題或外掛可能已經定義了更強的 CSS 權重。`!important` 是一個強制覆寫的語法，在這種除錯情境下非常有用。但請謹慎使用，不要濫用它，把它當成**最後的武器**。

## 總結與常見陷阱

恭喜你！你已經學會了專業的 RWD 除錯流程。

**重點回顧：**
1.  **SOP:** 使用開發者工具的「響應式模式」->「選取元素」->「即時修改樣式」->「撰寫 Media Query」。
2.  **優先級：** 永遠先嘗試使用 **Kadence Blocks** 或 **Elementor** 的內建響應式設定來調整。
3.  **最終手段：** 當內建設定無效時，才在「**外觀 > 自訂 > 附加的 CSS**」中加入你的 Media Query 程式碼。
4.  **快取問題：** 如果你修改了 CSS 卻看不到效果，**90% 的可能是快取問題**。請記得清除 **WP Rocket** 的快取，以及你瀏覽器本身的快取，然後再重新整理頁面。

掌握這個技能後，你將不再害怕任何 RWD 的挑戰，能夠充滿信心地打造在任何裝置上都表現完美的網站。
