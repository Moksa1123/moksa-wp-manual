# 03.03 (實務) 瀏覽器開發者工具 (F12) 除錯

嗨，歡迎來到 MoksaWP 的進階客製化課程！我是你的導師。

在我們深入探討 CSS 客製化、動態效果和網站優化的世界之前，我們必須先學會如何使用所有網頁設計師和開發者的**最強大的秘密武器**——**瀏覽器開發者工具 (Browser Developer Tools)**。

你可能聽過它，或是不小心按到 F12 鍵叫出過它，然後覺得那堆程式碼很可怕就關掉了。但請相信我，學會使用它，就像是給了你一副「X光眼鏡」，能夠透視任何網站的結構、樣式和運作方式。這堂課，我們將把這個看似複雜的工具，變成你解決問題、提升效率的最佳夥伴。

---

### 什麼是瀏覽器開發者工具？

簡單來說，它是內建在所有現代瀏覽器（如 Chrome, Firefox, Edge）中的一套工具，可以讓你即時**檢查、編輯和除錯**網站的 HTML、CSS 和 JavaScript。

**Moksa 技巧**：對於 WordPress 網站來說，**90% 的前端視覺問題**（例如：顏色不對、排版跑掉、按鈕沒反應）都可以透過開發者工具快速定位問題所在，甚至在正式修改前，直接在瀏覽器裡「預覽」修改後的效果！

### 如何開啟開發者工具？

有三種常用方式，選擇你最順手的一種即可：

1.  **萬用快速鍵：** 在鍵盤上按下 `F12` 鍵。
2.  **右鍵選單：** 在網頁任何地方點擊滑鼠右鍵，選擇「**檢查 (Inspect)**」。
3.  **組合快速鍵：**
    *   Windows: `Ctrl + Shift + I`
    *   Mac: `Cmd + Opt + I`

``
*截圖：一個網頁上，滑鼠右鍵選單中「檢查」選項被反白高亮的畫面。*
``

打開後，你會看到一個新的面板出現在你的瀏覽器視窗底部或右側。別被滿滿的程式碼嚇到，我們會逐一拆解最重要的幾個部分。

---

### 核心功能面板介紹

開發者工具有很多分頁（面板），但我們初期只需要專注在最實用的四個上面：

1.  **元素 (Elements):** 你的 CSS 視覺化編輯器。
2.  **主控台 (Console):** 你的 JavaScript 偵錯助手。
3.  **網路 (Network):** 你的網站效能分析儀。
4.  **應用程式 (Application):** 清理網站暫存的萬能鑰匙。

讓我們透過實務操作來學習它們。

### 面板一：元素 (Elements) - 你的 CSS 視覺化編輯器

這是你最常用到的面板。左邊是網站的 **HTML 結構**，右邊是對應元素的 **CSS 樣式**。

#### 實作練習 1：找出並修改 WooCommerce 商品按鈕的顏色

假設我們覺得預設的「加入購物車」按鈕顏色不好看，想換個顏色。傳統做法是回到 WordPress 後台，在佈景主題或頁面編輯器設定裡大海撈針。現在，我們用更專業的方式來處理。

*   **步驟 1：選取目標元素**
    *   在你的商品頁面上，對著「加入購物車」按鈕點擊**滑鼠右鍵**，選擇「**檢查**」。
    *   你會發現開發者工具的「Elements」面板會自動跳轉，並**反白高亮**對應這個按鈕的 HTML 程式碼 (通常是一個 `<button>` 或 `<a>` 標籤)。
    *   你也可以點擊開發者工具左上角的「**箭頭圖示 (Select an element)**」，然後再用滑鼠去點擊網頁上的任何元素，也能達到同樣的效果。

    ``
    *截圖：開發者工具畫面，左側 HTML 區域反白高亮了按鈕的程式碼，同時網頁上的按鈕也被藍色區塊覆蓋。*
    ``

*   **步驟 2：在樣式窗格中即時編輯 CSS**
    *   眼睛看到右邊的「**樣式 (Styles)**」窗格。這裡列出了所有套用在這個按鈕上的 CSS 規則。
    *   往下捲動，找到跟 `background-color` (背景顏色) 相關的設定。
    *   直接用滑鼠點擊顏色代碼 (例如 `#4f4f4f`)，會跳出一個調色盤。試著把它改成你想要的顏色，例如 Moksa 品牌色 `#7A5EFF`。

    ``
    *截圖：開發者工具右側的 Styles 窗格，滑鼠正在點擊 background-color 的色塊，並跳出了調色盤。網頁上的按鈕顏色也即時變成了新的顏色。*
    ``

*   **步驟 3：複製並儲存你的修改**
    *   **重要觀念**：你在開發者工具裡做的任何修改**都只是暫時的**，重新整理頁面後就會消失。這讓我們可以 без壓力地任意測試。
    *   當你調整到滿意的樣式後，我們需要把這段 CSS 程式碼複製出來。
    *   在剛剛修改的 CSS 規則區塊，找到它的**選擇器 (Selector)**，例如 `.woocommerce a.button`。
    *   **Moksa 技巧**：將這段完整的 CSS 規則（選擇器 + 大括號 + 你修改的屬性）複製下來。

    ```css
    .woocommerce a.button {
        background-color: #7A5EFF; /* 這是你修改後的新顏色 */
    }
    ```

*   **步驟 4：貼到 WordPress 的「附加的 CSS」中**
    *   回到 WordPress 後台，前往「**外觀**」>「**自訂**」。
    *   在自訂面板中，找到「**附加的 CSS (Additional CSS)**」。
    *   將你剛剛複製的程式碼貼進去，然後點擊「**發佈**」。

    ``
    *截圖：WordPress 後台的「自訂」>「附加的 CSS」畫面，程式碼被貼在輸入框中。*
    ``

恭喜！你已經學會了專業的 CSS 除錯與修改流程。這個技巧可以用來修改網站上**任何元素的任何樣式**。

### 面板二：主控台 (Console) - 你的 JavaScript 偵錯助手

當網站上的某些互動功能失靈時（例如：圖片輪播不動了、表單送不出去、彈出視窗沒反應），第一個要檢查的地方就是 Console。

*   **用途：**
    *   顯示由 JavaScript 產生的**錯誤訊息 (紅色)**、**警告 (黃色)** 或一般資訊。
    *   開發者可以在此執行 JavaScript 程式碼來進行測試。

*   **實務情境：**
    *   你安裝了兩個外掛，結果發現 FluentCRM 的彈出式訂閱表單突然無法正常顯示。
    *   **除錯步驟：**
        1.  開啟開發者工具，切換到「**Console**」面板。
        2.  重新整理頁面。
        3.  檢查是否有**紅色的錯誤訊息**出現。錯誤訊息通常會告訴你哪一個 JavaScript 檔案的第幾行出了問題。
        4.  **Moksa 技巧**：錯誤訊息中出現的外掛名稱（例如 `.../wp-content/plugins/some-other-plugin/script.js`）通常就是**衝突的來源**。這能幫助你快速判斷是不是外掛衝突，並回報給外掛開發者。

``
*截圖：Console 面板顯示一條紅色的 JavaScript 錯誤訊息，例如 "Uncaught TypeError: $ is not a function"。*
``

### 面板三：網路 (Network) - 你的網站效能分析儀

這個面板會記錄下瀏覽器為了載入你的網頁，所有發出的網路請求，包含圖片、CSS 檔案、JS 檔案等等。它是網站**效能優化**的關鍵工具。

#### 實作練習：檢查 WP Rocket 快取是否生效

我們使用 WP Rocket 來為網站加速，但你怎麼知道它真的有在運作呢？

*   **步驟 1：** 使用**無痕模式**或**登出**你的 WordPress 帳號（因為快取通常不會對登入的管理員生效）。
*   **步驟 2：** 開啟開發者工具，切換到「**Network**」面板。
*   **步驟 3：** 載入你的網站首頁。
*   **步驟 4：** 在請求列表中，點擊第一個請求（通常是你的網域名稱，Type 為 `document`）。
*   **步驟 5：** 在右邊的視窗中，切換到「**標頭 (Headers)**」分頁。
*   **步驟 6：** 往下找到「**回應標頭 (Response Headers)**」區塊。
*   **Moksa 技巧**：如果你看到 `x-wp-rocket-cache: hit` 這一行，就代表 WP Rocket 的頁面快取成功運作了！如果看到 `miss`，代表這次請求沒有命中快取，需要再重新整理一次。

``
*截圖：Network 面板，選中了主文件請求，右側的 Headers 區塊中，用紅框圈出了 "x-wp-rocket-cache: hit"。*
``

#### 其他用途：
*   **找出大型檔案：** 在 Network 面板中，點擊「**Size**」欄位來排序，你可以馬上找出是哪張圖片或哪個檔案拖慢了你的網站載入速度，然後使用 ShortPixel 進行優化。
*   **分析 AJAX：** 當 WooCommerce 的「加入購物車」或篩選器功能出錯時，可以篩選 `XHR` 請求，查看 `admin-ajax.php` 的回傳內容來找出錯誤原因。

### 面板四：應用程式 (Application) - 清理網站暫存的萬能鑰匙

這個面板儲存了瀏覽器為特定網站保留的資料，如 Cookies、快取檔案等。

*   **Moksa 技巧**：當你遇到一些奇怪的「快取」問題，例如你明明改了 CSS 但網站就是沒變、或是登入狀態異常時，傳統的 `Ctrl+F5` 強制重新整理可能還不夠。這時你可以用 Application 面板來個**徹底大掃除**。

*   **操作步驟：**
    1.  開啟開發者工具，切換到「**Application**」面板。
    2.  在左邊的選單中，點擊「**儲存空間 (Storage)**」。
    3.  點擊「**清除網站資料 (Clear site data)**」按鈕。
    4.  這會清除掉跟這個網站相關的所有 Cookies、快取、和本機儲存資料，效果比瀏覽器設定裡的「清除快取」更為徹底且精準。

``
*截圖：Application 面板的 Storage 頁面，滑鼠指標正準備點擊 "Clear site data" 按鈕。*
``

### 總結

瀏覽器開發者工具 (F12) 是你從網站使用者**蛻變為網站管理與設計專家的關鍵一步**。它將「我猜可能是...」的除錯方式，轉變為「我看到問題在...」的精準診斷。

剛開始可能會覺得資訊量很大，但請務必養成習慣：**遇到前端問題，就先按下 F12**。從檢查 Elements 的 CSS 開始，你會發現解決問題的能力和效率都將大幅提升。

在接下來的課程中，我們將會頻繁地使用到這個工具，請務必熟悉它。
