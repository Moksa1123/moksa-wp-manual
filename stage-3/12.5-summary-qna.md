## 課程核心回顧

在這 50 小時的課程中，我們深入探討了 WordPress 開發的四大核心支柱：

### 1. 專業開發環境與流程 (Workflow)
我們拋棄了傳統 FTP 上傳的低效率模式，建立了一套現代化的開發流程。
*   **本機開發環境:** 使用 **LocalWP** 快速建立與管理獨立的 WordPress 網站，告別網路延遲。
*   **程式碼編輯器:** 掌握 **VS Code** 的使用，包含常用快捷鍵、延伸模組 (Extensions) 的安裝與設定。
*   **版本控制:** 學習使用 **Git** 來管理你的程式碼，這是團隊協作與專案管理的基石。你現在應該能夠熟練使用 `git commit`, `git push`, `git pull` 等基本指令。
*   **指令列工具:** 體驗 **WP-CLI** 的強大之處，學會用指令快速安裝 WordPress、管理外掛/佈景主題、操作資料庫，大幅提升開發效率。

### 2. 佈景主題 (Theme) 開發實務
你已經從零到一，理解了一個 WordPress 佈景主題是如何運作的。
*   **核心架構:** 深入理解 `style.css` 檔案頭、`index.php` 與 `functions.php` 的作用。
*   **範本層級 (Template Hierarchy):** 掌握 WordPress 如何根據不同頁面（首頁、文章頁、分類頁）來決定載入哪個範本檔案 (`front-page.php`, `single.php`, `archive.php` 等)。
*   **The Loop:** 精通 WordPress 最核心的「迴圈」，並學會使用 `have_posts()`, `the_post()` 等函式來顯示文章資料。
*   **函式與載入:** 學習使用 `get_header()`, `get_footer()`, `get_sidebar()` 來組織程式碼，並透過 `wp_enqueue_script()` 與 `wp_enqueue_style()` **正確地**載入 CSS 與 JavaScript 檔案。

### 3. 外掛 (Plugin) 開發實務
你學會了如何將特定功能模組化，開發出屬於你自己的外掛。
*   **外掛基礎:** 了解外掛的檔案結構、註解頭資訊，以及啟動 (`activation`) 與停用 (`deactivation`) 的處理。
*   **Hooks (掛鉤):** 這是 WordPress 的精髓！你已經能夠分辨並善用 **Actions (動作)** 與 **Filters (過濾器)** 來掛載自己的功能，而不需要修改核心檔案。
*   **自訂內容類型 (CPT & Taxonomy):** 使用 `register_post_type()` 和 `register_taxonomy()` 創建自訂的文章類型（例如：作品集、產品）與分類。
*   **整合 ACF (Advanced Custom Fields):** **Moksa 技巧**，我們學習了如何結合 ACF 的強大欄位功能與我們的自訂外掛，讓後台資料管理變得既彈性又直觀。

### 4. 進階除錯與 API 整合
開發過程中，問題在所難免，你已經學會如何有效率地找出並解決問題。
*   **除錯工具:** 學會開啟 `WP_DEBUG` 模式，並使用 **Query Monitor** 外掛來分析頁面效能、資料庫查詢與 PHP 錯誤。
*   **REST API:** 初步認識了 WordPress 內建的 REST API，了解如何透過 HTTP 請求來讀取或寫入 WordPress 資料，為前後端分離或 App 開發打下基礎。
*   **Gutenberg (區塊編輯器) 開發:** 我們淺嚐了使用 React.js 開發自訂區塊的過程，讓你了解 WordPress 未來的主流開發趨勢。

## 常見問題 (Q&A)

這裡是我們整理過往學員在課程結束後，最常提出的幾個問題。

### Q1: 我應該選擇自己開發佈景主題，還是使用像 Kadence 這樣的頁面編輯器主題？

這是一個非常好的問題，答案取決於**專案的需求、預算與時程**。

*   **使用 Kadence Blocks / Elementor:**
    *   **優點:** 開發速度極快、客戶可以自行修改大部分版面、生態系完整（有大量第三方擴充）。
    *   **適用情境:** 企業形象網站、內容為主的部落格、大部分的 WooCommerce 電商網站。**Moksa 技巧**：對於 90% 的專案來說，使用高品質的頁面編輯器搭配少量客製化 CSS/PHP，是**成本效益最高**的選擇。

*   **從頭開發佈景主題:**
    *   **優點:** 程式碼極致乾淨、效能最佳化、完全客製化的後台體驗與前端互動。
    *   **適用情境:** 對效能有極端要求的網站、需要高度整合第三方系統的 Web App、功能與版面非常獨特的專案、SaaS 服務。

**總結：** 先精通 Kadence，用它來完成大部分的案子。當你遇到 Kadence 無法滿足的深度客製化需求時，再動用你所學的佈景主題開發技能。

### Q2: 我寫的程式碼出錯了，畫面全白，該怎麼辦？

畫面全白 (White Screen of Death) 是開發者必經的過程。請依照以下步驟除錯：

*   **步驟 1：開啟除錯模式**
    *   打開 `wp-config.php` 檔案，找到 `WP_DEBUG` 並將其值改為 `true`。
    *   ```php
      define( 'WP_DEBUG', true );
      define( 'WP_DEBUG_LOG', true ); // 將錯誤寫入 /wp-content/debug.log 檔案
      define( 'WP_DEBUG_DISPLAY', true ); // 將錯誤顯示在畫面上
      ```
    *   重新整理頁面，通常畫面上就會顯示詳細的 PHP 錯誤訊息，告訴你哪一個檔案的第幾行出錯。

*   **步驟 2：使用 Query Monitor**
    *   如果網站前後台還能進入，安裝並啟用 **Query Monitor**。它會在頂部管理列顯示詳細的 PHP Errors / Warnings，比原生的錯誤訊息更容易閱讀。
    ``

*   **步驟 3：逐一排除**
    *   如果是因為某個外掛造成的，可以先透過 FTP 或檔案總管，將 `/wp-content/plugins/` 裡面的可疑外掛資料夾改名，強制停用它。
    *   回想你最近修改了哪個檔案的程式碼，通常問題就出在那裡。

### Q3: 客戶網站需要加上我寫的客製化功能，我應該寫在 `functions.php` 還是做成一個小外掛？

**Moksa 技巧：永遠優先選擇「做成一個小外掛」。**

*   **寫在 `functions.php` 的缺點:**
    *   當客戶未來更換佈景主題時，你寫的所有功能都會**全部遺失**。
    *   `functions.php` 容易變得臃腫不堪，難以維護。
    *   不同功能的程式碼混雜在一起。

*   **做成外掛的優點:**
    *   **功能與外觀分離：** 這是 WordPress 開發的核心原則。功能應該由外掛掌管，外觀由佈景主題負責。
    *   **可攜性高：** 更換佈景主題不影響功能。
    *   **易於管理：** 你可以隨時啟用或停用這個功能，方便除錯。
    *   **程式碼清晰：** 一個外掛專注於一件事情。

你可以為客戶建立一個名為 `[客戶名稱]-functionality` 的專屬外掛，將所有客製化程式碼（例如 CPT 註冊、客製化的 WooCommerce 邏輯）都放進去。

### Q4: 我該如何持續精進我的 WordPress 開發技能？

*   **閱讀官方文件:** [WordPress Developer Resources](https://developer.wordpress.org/) 是你的聖經，所有函式、Hooks 的用法都可以在這裡找到最權威的解釋。
*   **逛 Stack Exchange:** [WordPress Development Stack Exchange](https://wordpress.stackexchange.com/) 有大量的實務問題與高品質的解答。
*   **追蹤大神:** 關注 WordPress 開發領域的專家部落格或 Twitter，例如 a11n, Tom McFarlin, Pippin Williamson (Pippins Plugins) 等。
*   **動手做專案:** 這是最重要的一點。嘗試複製一個你喜歡的網站功能、為自己寫一個實用的小外掛、或是到 GitHub 上參與開源專案的貢獻。
*   **持續關注 Moksa 的進階課程與社群：** 我們會持續推出更深入的課程，並在社群中分享最新的開發趨勢與技巧。
