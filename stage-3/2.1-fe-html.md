# 02.01 語意化 HTML5

嗨，歡迎來到 MoksaWP 佈景主題與外掛開發課程的第二章。在我們深入探討 PHP、JavaScript 與 WordPress 複雜的 API 之前，我們必須先打好最穩固的地基——**HTML (HyperText Markup Language)**。

更具體地說，我們要學習的是 **HTML5 的「語意化 (Semantic)」寫法**。

想像一下，你在蓋一棟房子。你可以用一堆長得一模一樣的磚塊（`<div>`）把所有牆壁都砌起來，從外面看，它或許像個房子。但當你走進去，你會分不清楚哪裡是客廳、哪裡是臥室、哪裡是廚房。

**語意化 HTML** 就是為你的「網頁房子」的不同區域，掛上清晰的標示牌。我們不再只用通用的 `<div>` 和 `<span>`，而是使用帶有**特定意義**的標籤，例如 `<header>` 代表頁首、`<nav>` 代表導覽列、`<main>` 代表主要內容區。

這堂課的目標，是讓你徹底理解為什麼語意化如此重要，並學會如何使用這些標籤來建構一個**結構清晰、對 SEO 友好、且易於維護**的 WordPress 網站。

## 為什麼語意化 HTML 如此重要？

在開始學習具體標籤之前，你必須先明白「為什麼」我們要這麼做。這不是為了標新立異，而是為了帶來實質的好處。

### 1. **搜尋引擎優化 (SEO)**
這是最直接、也是最重要的原因之一。Google、Bing 等搜尋引擎的爬蟲 (Crawler) 在讀取你的網頁時，它們不是用「眼睛」看的，而是透過**解析 HTML 結構**來理解內容。

*   當你使用 `<h1>`，你就是在告訴搜尋引擎：「這是這頁最重要的標題」。
*   當你使用 `<nav>`，你是在說：「這裡面是網站的主要導覽連結」。
*   當你使用 `<article>`，你是在標示：「這是一篇獨立、完整的內容」。

一個語意清晰的網站，能讓搜尋引擎更精準地理解你的網頁架構與內容重點，從而給予更高的排名。這也是 **Moksa 技巧** 中，我們如此強調使用 **Rank Math SEO** 外掛進行設定後，仍需從根本的程式碼結構去優化的原因。

### 2. **無障礙 (Accessibility, a11y)**
世界上有許多使用者需要依賴輔助科技（例如螢幕閱讀器）來瀏覽網頁。對於視障者而言，螢幕閱讀器會根據 HTML 標籤來「朗讀」網頁內容。

如果你的網頁充滿了無意義的 `<div>`，螢幕閱讀器只能從頭到尾平鋪直敘地念下去，使用者會迷失方向。但如果你使用了語意化標籤，螢幕閱讀器就可以告訴使用者：

*   「現在你正在『導覽列』區域，你可以按 Tab 鍵跳到下一個連結。」
*   「現在你進入了『主要內容』區域。」
*   「這是一篇文章的『頁尾』，包含了作者資訊和發佈日期。」

打造一個無障礙的網站，不僅是專業開發者的責任，也是擴大潛在客戶群的商業考量。

### 3. **程式碼可維護性**
當你或你的團隊成員在幾個月後需要回頭修改網站時，一個語意化的 HTML 檔案會像一份清晰的藍圖。

**糟糕的例子 (Non-Semantic):**
```html
<div id="header">...</div>
<div class="main-content">
  <div class="post">...</div>
</div>
<div id="footer">...</div>
```
你需要透過 `id` 或 `class` 才能猜測每個區塊的用途。

**良好的例子 (Semantic):**
```html
<header>...</header>
<main>
  <article>...</article>
</main>
<footer>...</footer>
```
程式碼不言自明，一目了然。這大大降低了維護成本，也讓 CSS 樣式的撰寫更加直觀。

## 常見的 HTML5 語意化區塊標籤

接下來，讓我們認識一下建構網頁結構時最常用的幾個語意化標籤。

``
*一張圖解釋各個語意化標籤在一個典型網頁佈局中的位置*

### `<header>`
代表整個網頁或某個區塊 (例如 `<article>`) 的**頁首**。通常包含網站 Logo、主標題、搜尋框等。

```html
<header>
  <a href="/" class="logo">MoksaWP</a>
  <h1>佈景主題與外掛開發課程</h1>
</header>
```

### `<nav>`
代表**導覽連結**的區塊。通常會包住一個無序清單 `<ul>` 來放置主要選單。

```html
<nav>
  <ul>
    <li><a href="/courses">所有課程</a></li>
    <li><a href="/blog">技術文章</a></li>
    <li><a href="/contact">聯絡我們</a></li>
  </ul>
</nav>
```

### `<main>`
代表網頁的**主要內容**。一個頁面**應該只有一個** `<main>` 標籤。所有與該頁面核心主題直接相關的內容都應該放在這裡。

```html
<main>
  <h2>歡迎學習語意化 HTML5</h2>
  <p>這是本章節的核心內容...</p>
</main>
```

### `<article>`
代表一份**獨立、完整且可被再次發佈**的內容。例如：一篇部落格文章、一則論壇貼文、一個商品項目。`<article>` 內部可以有自己的 `<header>` 和 `<footer>`。

```html
<article>
  <header>
    <h2>如何選擇 WordPress 主機？</h2>
    <p>發佈於：2023-10-27</p>
  </header>
  <p>選擇主機是建立網站的第一步...</p>
  <footer>
    <p>作者：Moksa 團隊</p>
  </footer>
</article>
```

### `<section>`
代表文件中一個**獨立的主題群組**。它通常會有一個標題 (e.g., `<h2>`)。選擇 `<section>` 的一個好方法是：如果這塊內容無法在 `<article>` 的語境下成立，但又可以被獨立切割出來並給予一個標題，那它就適合用 `<section>`。例如：首頁的「關於我們」、「最新消息」、「客戶見證」等區塊。

```html
<section>
  <h2>客戶見證</h2>
  <blockquote>"Moksa 的課程讓我受益匪淺！"</blockquote>
  <blockquote>"技術支援非常到位！"</blockquote>
</section>
```

### `<aside>`
代表與主要內容**相關性較低**的補充內容。通常被用作側邊欄 (Sidebar)，放置相關文章、廣告、作者簡介等。

```html
<aside>
  <h3>相關文章</h3>
  <ul>
    <li><a href="#">什麼是 Child Theme？</a></li>
    <li><a href="#">10 個必備的 WordPress 外掛</a></li>
  </ul>
</aside>
```

### `<footer>`
代表整個網頁或某個區塊的**頁尾**。通常包含版權資訊、聯絡方式、隱私權政策連結等。

```html
<footer>
  <p>&copy; 2024 MoksaWeb. All rights reserved.</p>
  <a href="/privacy-policy">隱私權政策</a>
</footer>
```

## 組合應用：一個 WordPress 文章頁面的典型結構

現在，讓我們把這些標籤組合起來，看看一個典型的 WordPress 文章頁面 (`single.php`) 的 HTML 結構會是什麼樣子。

```html
<!DOCTYPE html>
<html lang="zh-TW">
<head>
  <meta charset="UTF-8">
  <title>我的文章標題 - MoksaWP</title>
</head>
<body>

  <!-- 通常對應到 header.php -->
  <header class="site-header">
    <div class="logo">...</div>
    <nav class="main-nav">...</nav>
  </header>

  <!-- 整個頁面的主要內容區 -->
  <main id="content" class="site-content">

    <!-- 這是一篇獨立的文章，對應到 a single post -->
    <article class="post">
      
      <header class="entry-header">
        <h1>文章主標題 (H1)</h1>
        <div class="entry-meta">發佈日期、作者...</div>
      </header>

      <div class="entry-content">
        <p>這是文章的第一段...</p>
        <figure>
          <img src="..." alt="...">
          <figcaption>圖片說明文字</figcaption>
        </figure>
        <p>這是文章的第二段...</p>
      </div>

      <footer class="entry-footer">
        <span class="categories">分類</span>
        <span class="tags">標籤</span>
      </footer>

    </article>

    <!-- 相關文章區塊 -->
    <section class="related-posts">
      <h2>相關文章</h2>
      ...
    </section>

  </main>

  <!-- 側邊欄，通常對應到 sidebar.php -->
  <aside class="sidebar">
    <h3>關於作者</h3>
    ...
  </aside>

  <!-- 網站全域頁尾，通常對應到 footer.php -->
  <footer class="site-footer">
    <p>版權資訊...</p>
  </footer>

</body>
</html>
```

## Moksa 實戰技巧

1.  **檢查頁面編輯器產生的 HTML**
    當你使用 **Kadence Blocks** 或 **Elementor** 這類頁面編輯器時，花點時間用瀏覽器的「開發者工具 (F12)」檢查它們產生的 HTML 結構。雖然它們很方便，但有時會為了達成視覺效果而產生過多層的 `<div>`。理解語意化能幫助你判斷何時該用編輯器，何時該用手寫程式碼來獲得更乾淨、更高效的結構。

2.  **ACF 與 CPT 的樣板檔案**
    當你在開發自訂文章類型 (Custom Post Type, CPT) 並搭配進階自訂欄位 (ACF) 時，你將會建立自己的樣板檔案 (e.g., `single-product.php`)。這正是你大展身手、實踐語意化 HTML 的最佳時機。**不要**只是用 `<div>` 把所有 ACF 欄位包起來，思考每個欄位內容的「意義」，並為它們選擇最合適的標籤。

3.  **`<article>` vs `<section>` 的抉擇**
    這是初學者最容易混淆的地方。**Moksa 技巧**：問自己一個問題：「**這塊內容如果獨立抽出來，放到 RSS feed 或另一個網站上，它還完整且有意義嗎？**」
    *   如果答案是「是」，那就用 `<article>` (e.g., 一篇部落格文章)。
    *   如果答案是「否」，那它可能只是一個頁面中的某個部分，用 `<section>` 更為合適 (e.g., 首頁上的「我們的服務」區塊)。

## 總結

語意化 HTML 是現代網頁開發的基石。它不僅僅是「寫對」程式碼，更是一種專業的開發思維。養成使用語意化標籤的習慣，你所打造的 WordPress 網站將會擁有更佳的 SEO 體質、更好的使用者體驗（特別是對於輔助科技使用者），以及更清爽、更容易維護的程式碼。

在接下來的課程中，我們將基於這個穩固的 HTML 結構，開始學習如何用 CSS 來美化它。
