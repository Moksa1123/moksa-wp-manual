# 6.2 (實作) 管理者的標準出貨流程 (SOP)

嗨，歡迎來到這個實作單元！當顧客完成付款後，身為網站管理員或倉管人員，最重要的一件事就是如何快速、正確地將商品送到顧客手中。一個順暢的出貨流程不僅能提升營運效率，更能大幅提高顧客滿意度。

在這個章節，我們將會一步步帶你走過一個基於 Moksa 技術棧的**標準化出貨流程 (SOP)**。這個流程整合了 WooCommerce 核心功能、RY Tools 超商物流、綠界電子發票，以及 LINE 訂單通知，確保你在處理訂單時，每一個環節都精準到位。

---

### **前置作業：確認訂單狀態**

在開始出貨流程前，我們必須先學會如何判斷哪些訂單是「可以出貨」的。

在 WordPress 後台，請前往 `WooCommerce` > `訂單`。你會看到所有訂單的列表。

``

我們要找的訂單狀態是「**處理中 (Processing)**」。

*   **為什麼是「處理中」？**
    *   當顧客使用**信用卡 (綠界 ECPay)** 或 **LINE Pay** 付款成功後，訂單狀態會自動變為「處理中」。這表示**款項已確認**，你可以安心出貨。
    *   如果訂單狀態是「**保留 (On Hold)**」，通常代表顧客選擇了 **ATM 轉帳**，但尚未完成付款。**請勿**在此階段出貨。
    *   如果訂單狀態是「**等待付款 (Pending payment)**」，代表顧客尚未選擇付款方式或付款失敗。

**Moksa 技巧：** 你可以利用訂單列表上方的篩選器，直接點擊「處理中」，快速找出所有已付款、等待出貨的訂單，讓你的工作台保持乾淨俐落。

---

### **出貨流程四大步驟 (SOP)**

現在，讓我們開始實際操作。請點進一筆狀態為「處理中」的訂單。

#### **步驟 1：產生超商寄件單 (使用 RY Tools)**

進入訂單詳細頁面後，你的目光請移到右側的工具欄，找到一個名為「**RY 7-11/全家 C2C 寄貨**」的區塊。這裡是我們處理超商物流的核心區域。

1.  **確認寄件資訊**：系統會自動帶入顧客在結帳時選擇的超商門市。請快速核對門市名稱是否正確。
2.  **建立寄貨訂單**：點擊「**建立 7-11/全家寄貨訂單**」按鈕。
    ``
3.  **取得寄件編號**：點擊後，RY Tools 會透過 API 與超商的物流系統溝通。成功後，這個區塊會更新，並顯示一組「**寄件編號 (Shipping Code)**」或「**取貨付款編號**」。
    ``

**Moksa 技巧：** 這個「寄件編號」非常重要，它是這件包裹在物流系統中的唯一識別碼。如果產生失敗，通常是超商門市暫停服務或地址資訊有誤，可以請顧客更換門市後再試一次。

#### **步驟 2：列印寄件標籤**

取得了寄件編號後，我們需要將它印出來，貼在包裹上。

1.  在同一個 RY Tools 區塊中，點擊「**列印寄貨單**」按鈕。
2.  系統會在新分頁開啟一個 PDF 檔案，這就是你的寄件標籤。
    ``
3.  使用印表機將它印出。

**Moksa 技巧：**
*   **專業化設備**：為了提升效率，強烈建議投資一台**熱感應標籤機 (例如：4x6 吋)**。這樣你就不需要使用 A4 紙列印，省去了裁剪和黏貼的麻煩，出貨速度會快上好幾倍。
*   **撿貨單**：在包裝商品前，可以先印出 WooCommerce 內建的「撿貨單」，對照訂單內容，確保商品品項與數量完全正確，降低出貨錯誤的機率。

#### **步驟 3：完成訂單並觸發自動化**

當你已經將商品包裝好，並貼上寄件標籤後，就可以回到訂單頁面，進行最後一個、也是**最關鍵的**一個步驟。

1.  在訂單詳細資料頁面的左上方，找到「**訂單狀態**」的下拉選單。
2.  將狀態從「處理中」改為「**已完成 (Completed)**」。
3.  點擊右側的「**更新**」按鈕儲存變更。

``

**這個「更新」按鈕按下去，會同時觸發三件重要的自動化任務：**

1.  **觸發 WooCommerce Email**：系統會自動寄送「訂單已完成」的 Email 通知信給顧客。信件中會包含物流追蹤的相關資訊。
2.  **觸發綠界電子發票**：我們通常會設定在訂單「已完成」時，才自動開立**綠界電子發票 (ECPay Invoice)**。這確保了你不會為尚未出貨或可能取消的訂單預先開發票。
3.  **觸發 LINE 訂單通知**：如果你安裝了 **Order Notify**，系統會在此時發送一則 LINE 通知給顧客，告訴他們：「您的訂單已出貨！」提供即時、安心的購物體驗。

#### **步驟 4：前往超商寄件**

這是流程的最後一哩路。

1.  將你所有包裝好的包裹帶到對應的超商 (7-11 或全家)。
2.  操作門市的機台 (ibon 或 FamiPort)，選擇「代碼輸入」或「掃描」，然後輸入或掃描你標籤上的寄件編號/條碼。
3.  機台會列印出一張收據。
4.  將包裹和收據交給店員，他們會掃描收據並收下包裹。

**Moksa 技巧：** 店員給你的那張**收據務必保留好**！這是你寄出包裹的唯一憑證。建議至少保留到顧客確認收到商品為止。

---

### **總結與最佳實務**

恭喜你！你已經學會了一套完整、高效且自動化的 WooCommerce 出貨流程。

讓我們快速回顧一下這個 SOP 的精髓：

1.  **篩選訂單**：只處理狀態為「**處理中**」的訂單。
2.  **產生單號**：進入訂單，使用 **RY Tools** 建立寄貨單，取得寄件編號。
3.  **列印標籤**：印出寄件標籤並黏貼至包裹上。
4.  **完成訂單**：將訂單狀態改為「**已完成**」，一次性觸發 Email、電子發票與 LINE 通知。
5.  **實體寄件**：將包裹帶到超商完成寄送。

記住，將訂單狀態改為「已完成」是整個流程的**自動化核心**。只要遵循這套 SOP，你就能輕鬆管理每日的出貨作業，並為你的顧客提供世界級的購物體驗。
