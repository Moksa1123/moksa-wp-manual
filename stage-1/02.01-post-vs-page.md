# 2.1 「文章」與「頁面」的差別

嗨，歡迎來到這一堂課！在我們正式開始打造網站內容之前，有一個 WordPress 最基礎、也最重要的核心觀念必須要徹底釐清，那就是「**文章 (Post)**」與「**頁面 (Page)**」的差別。

很多新手會把這兩者搞混，覺得它們看起來都是一個可以打字、放圖片的地方，有什麼不同呢？事實上，它們的設計目的、組織方式與在網站上扮演的角色截然不同。搞懂它們的差異，將會是你建立一個**結構清晰、使用者體驗良好、且有利於 SEO** 的網站的基石。

我們可以先用一個簡單的比喻來理解：

*   **文章 (Post)** 就像是**雜誌裡的「專欄文章」或「新聞報導」**。它們有時效性，會隨著時間不斷增加，並且通常會按照日期新舊排序。
*   **頁面 (Page)** 就像是**雜誌的「版權頁」、「關於我們」或「目錄」**。它們是固定、永恆的內容，不會經常變動，是構成整本雜誌骨架的部分。

接下來，我們將深入探討它們的具體差異與使用情境。

### 重點速覽：文章 vs. 頁面 比較表

| 特性 | 文章 (Post) | 頁面 (Page) |
| :--- | :--- | :--- |
| **核心目的** | 具時效性的內容，如新聞、部落格、公告 | 靜態、永恆的內容，如「關於我們」 |
| **時效性** | **高** (有發佈日期) | **低** (通常不顯示日期) |
| **組織方式** | **分類 (Categories)** 與 **標籤 (Tags)** | **父子層級 (Parent-Child)** |
| **預設顯示位置** | 部落格、最新消息列表 (依時間排序) | 網站主選單、頁首/頁尾連結 |
| **作者與日期** | 通常會顯示 | 通常不顯示 |
| **留言功能** | 預設開啟 | 預設關閉 |
| **RSS Feed** | **會**被包含在 RSS Feed 中 | **不會**被包含在 RSS Feed 中 |

## 一、文章 (Post)：具時效性的內容串流

**文章**是 WordPress 的靈魂，它是為了「**動態內容**」而生。當你想要發佈具有時間性的資訊，例如公司最新消息、產品開箱文、產業知識分享、特價活動公告等，就應該使用「文章」。

### 文章的核心組織工具：分類與標籤

為了管理大量且不斷增加的文章，WordPress 提供了兩種強大的分類工具：

1.  **分類 (Categories):**
    *   **用途：** 用於文章的**主要、廣泛性分組**，就像是書本的「章節」。分類是**有層級關係的**，你可以建立「父分類」與「子分類」。
    *   **範例：** 一個咖啡電商的部落格，分類可以設定為「咖啡知識」、「沖煮技巧」、「品牌故事」。在「沖煮技巧」底下，還可以有「手沖」、「義式濃縮」等子分類。
    *   **Moksa 技巧：** 一篇文章**至少要屬於一個分類**，但建議不要超過兩個。這有助於保持網站結構清晰。

2.  **標籤 (Tags):**
    *   **用途：** 用於描述文章的**特定關鍵字**，就像是書本最後的「索引」。標籤是**沒有層級關係的**。
    *   **範例：** 在一篇介紹「手沖」的文章中，你可以加上「V60」、「KONO」、「水溫」、「粉水比」等標籤。使用者點擊「V60」這個標籤，就可以看到所有與 V60 相關的文章，無論它們屬於哪個分類。
    *   **Moksa 技巧：** 我們建議先規劃好你的**核心「分類」架構**，再針對每篇文章的細節內容自由加上「標籤」。一個好的分類架構對 SEO 至關重要。

## 二、頁面 (Page)：獨立且永恆的網站骨架

**頁面**是用來建立網站的「**靜態內容**」與「**核心架構**」。這些內容通常是訪客認識你的品牌、尋找重要資訊的地方，它們的內容相對固定，不太會隨時間改變。

### 頁面的核心組織工具：父子層級

頁面不像文章有分類和標籤，它唯一的組織方式是**屬性 (Attributes)** 中的「**父層 (Parent)**」設定。

*   **用途：** 透過設定父層頁面，你可以建立出網站的**層級結構**。
*   **範例：**
    *   你可以建立一個名為「關於我們」的**父層頁面**。
    *   然後再建立「我們的團隊」、「品牌故事」、「媒體報導」等**子層頁面**，並將它們的父層都設定為「關於我們」。
    *   這樣在網址上可能會呈現 `yourdomain.com/about-us/our-team/` 的結構，非常有條理。

常見的頁面應用包括：
*   **首頁 (Homepage)**
*   **關於我們 (About Us)**
*   **聯絡我們 (Contact Us)**
*   **服務項目 (Services)**
*   **隱私權政策 (Privacy Policy)**
*   **購物說明 (Shopping Guide)**
*   **常見問答 (FAQ)**

## 三、我該如何選擇？情境式思考

當你不確定該用文章還是頁面時，可以問自己以下幾個問題：

*   **這份內容會過期嗎？或者它跟特定日期有關嗎？**
    *   是 -> **使用文章** (例如：2024 母親節特賣活動)
    *   否 -> **使用頁面** (例如：退換貨政策)

*   **我希望這份內容出現在「最新消息」列表裡嗎？**
    *   是 -> **使用文章**
    *   否 -> **使用頁面**

*   **這份內容是網站的核心資訊，適合放在主選單讓訪客隨時點擊嗎？**
    *   是 -> **使用頁面** (例如：聯絡我們)
    *   否 -> **使用文章** (例如：一篇關於如何清潔咖啡機的教學)

## 四、Moksa 技巧：從營運與 SEO 角度看文章與頁面

### 1. SEO 觀點 (Rank Math)

*   **頁面 (Page) 適合做「基石內容 (Pillar Content)」:**
    對於你的核心業務，可以建立一個內容非常豐富、深入的「頁面」，例如「新手手沖咖啡終極指南」。這個頁面是你的 SEO 基石。
*   **文章 (Post) 用來建立「主題簇 (Topic Cluster)」:**
    接著，你可以撰寫許多篇「文章」，例如「V60 濾杯沖煮技巧」、「咖啡豆研磨度解析」、「不同水溫對風味的影響」，這些文章都從不同角度探討主題，並**全部都連結回你的「新手手沖咖啡終極指南」頁面**。
*   這種「Pillar-Cluster」的策略，能有效地向 Google 證明你在這個領域的**專業性與權威度**，是 Rank Math SEO 中非常推崇的策略。

### 2. WooCommerce 網站特別提醒

當你安裝了 WooCommerce，你會在後台看到一個新的項目叫做「**商品 (Products)**」。請特別注意：

*   「**商品**」既不是文章，也不是頁面。它是一種 WordPress 術語中稱為「**自訂內容類型 (Custom Post Type, CPT)**」的東西。
*   你可以把「商品」想像成一種**特別的文章**，它擁有自己的「商品分類」、「商品標籤」，還有價格、庫存等專屬欄位。
*   **Moksa 技巧：** **千萬不要**用「文章」或「頁面」來上架你的商品。務必使用 WooCommerce 提供的「**商品**」功能來建立你的銷售品項，這樣才能搭配金流、物流與庫存管理系統。

### 總結

簡單來說，**用「頁面」來搭建你網站的骨架，用「文章」來為你的網站注入源源不絕的活水與血肉**。

在接下來的課程中，我們將會大量實作如何使用 **Kadence Blocks** 或 **Elementor** 來編輯「文章」與「頁面」，並將它們打造成專業又美觀的樣貌。今天，請務必先將這兩者的核心差異牢記在心！
